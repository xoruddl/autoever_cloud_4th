## Comprehension
1) 개요
- 반복 가능한 객체들을 이용해서 새로운 반복 가능한 객체를 축약한 형태로 생성하는 방법
- list, tuple, set, dict 등을 이용
- 반복 가능한 객체들을 간소화하게 되면 코드 구성이 깔끔해서 더 읽기 쉬운 코드가 만들어짐

2) 예시
```
l = list(range(1, 11))
result = []
for i in l:
    result.append(i * i)

print(result)

# Comprehension 이용
reuslt = [i * i for i in l]
print(result)

result = [i * i for i in range(1, 11) if i % 2 == 0]
print(result)

# 2개의 시퀀스를 사용하는 것도 가능
result = [i + j for i in range(1, 11) for j in range(101, 111)]
print(result)

# 2차원 배열 활용
l = [[1, 2, 3], [4, 5, 6]]
result = [j for i in l for j in i]
print(result)

# 1부터 10까지의 숫자에서 짝수이면 제곱을 하고 홀수이면 그대로
result = [i * i if i % 2 == 0 else i for i in range(1, 11)]
print(result)
```

# 7. Exception Handling
## 오류
1) 오류의 종류
- 컴파일 에러: 문법적인 오류로 인해서 실행이 되지 않는 경우
- 논리적 에러: 에러가 발생하지는 않지만 알고리즘의 오류로 인해 잘못된 결과가 만들어지는 경우
- exception(예외): 문법적인 에러는 없어서 실행이 되지만 외부 요인이나 잘못된 입력 등으로 실행 도중 중지되는 경우
- assertion(단언): 의도적으로 예외를 발생시키는 것

1) 오류 발생 시 조치
- 컴파일 에러가 발생하면 에러를 수정해서 실행: 컴파일 오류는 대다수의 IDE가 표시를 해주기 떄문에 비교적 찾기 쉬움
- 논리적 에러나 예외는 디버깅을 통해서 메모리를 확인해나가면서 수정하거나 예외처리 구문을 추가

## Exception (예외)
- 문법적으로는 이상이 없어서 컴파일 시점에는 아무런 문제가 없지만 프로그램 실행 도중에 발생하는 예기치 않은 사건으로 프로그램이 중단되는 현상
- 예외가 발생하는 경우
    - 정수를 0으로 나누는 경우
    - 파일을 읽고자 하는데 파일이 없는 경우
    - 통신을 하고자 하는데 상대방이 접속되지 않은 경우

- 예외 처리는 예외가 발생했을 때 수행될 작업을 작성하는 것

- 예외 처리의 목적
    - 정상 종료
        - 외부 자원을 사용하는 경우 예외가 발생하면 정상적으로 자원을 반납하고 종료하도록 작성
    - 예외 내용을 기록
        - 다음에 이러한 예외가 발생하지 않도록 하기 위해서
    - 무시하고 계속 실행
        - 예외가 그리 치명적이지 않아서 무시하고 실행
    - 값을 변경해서 정상적으로 수행
        - 비정상적인 값이 입력되는 경우 정상적인 값으로 변경해서 수행

- 기본적인 예외 처리
```
try: 
    예외가 발생할 가능성이 있는 문장
except:
    예외가 발생했을 때 처리할 문장
```

- except 다음에 예외 종류를 설정해서 특정 예외만 처리할 수 있으며 여러개의 except를 만들 수 있다.
    - 여러 개의 except를 만드는 경우 상위 클래스의 except 클래스를 위에 작성하면 안됨
    - 객체지향 프로그래밍에서는 사우이 클래스 타입의 변수에 하위 클래스 타입의 인스턴스를 대입할 수 있다.
    - 상단에 사우이 클래스 타입의 예외를 처리하는 클래스를 두게 되면 위에서 처리하기 떄문에 아래 구문은 Dead Code가 된다.
    - 파이선에서 예외 처리의 최상위 클래스는 BaseException -> Exception

```
try:
    x = int(input("정수 입력: "))
    y = 10 / x
    print(y)
except ValueError:
    print("정수가 아닌 데이터가 입력됨")
except ZeroDivisionError:
    print("0으로 나눌 수 있겠노")
except Exception:
    print("0 문제 아니고 정수 문제도 아님 근데 예외발생")
else:
    print("예외가 발생하지 않은 경우 수행할 문장")
finally:
    print("무조건 수행될 문장")
```

- else 와 finally
    - try 다음에 else 블럭을 추가하면 예외가 발생하지 않은 경우 수행될 코드를 작성할 수 있다.
    - try 다음에 finally 블럭을 추가하면 예외 발생 여부와 상관없이 수행될 문장을 작성할 수 있다. 


- 강제로 예외 발생
    - raise 예외 클래스(메시지) 구문을 호출하면 강제로 예외를 발생시킬 수 있다.

## 단언
- assert를 이용하면 특정 조건을 만족하지 않을 때 AssertionError를 발생시킬 수 있다.

```
try:
    x = int(input("정수 입력: "))
    # x가 11 이상이면 AssertionError 발생
    assert x < 11
    y = 10 / x
    print(y)
except AssertionError:
    print("단언을 이용해서 예외를 발생시킴")
```

    - 파일객체 = open()

## open
- 파일에 기록하기
    - 파일에 기록을 할 때는 2개나 3개의 매개변수를 대입
    - 첫번쨰 매개변수는 파일의 경로인데 없으면 만들어줌
    - 두번째 매개변수는 모드로 'w'를 설정함
    - 세번째 매개변수는 인코딩 방식인데 생략하면 현재 출력 인코딩이 설정됨
    - 파일에 기록을 할 때는 write 함수를 호출해서 기록할 데이터를 넘겨주면 됨
    - \n이 포함된 문자열의 컬렉션의 경우는 writeness 나 join을 이용해서 결합한 후 한 번에 기록하는 것도 가능

- 파일의 내용 읽어오기 
    - 파일의 내용을 읽어올 때는 파일의 경로만 대입하면 읽어옴.
    - 인코딩 방식을 추가로 설정할 수 있다.
    - 전체 데이터를 하나의 문자열로 읽고자하는 경우에는 read()를 호출하면 됨
    - 줄 단위로 읽고자 할 때는 파일 객체의 반복자나 readline 또는 readlines를 이용해서 list로 만들 수 있다.
    - read(바이트 수)를 이용해서 특정 바이트만큼 읽어 올 수 있다.

- 경로
    - 절대 경로
        - 루트로부터의 경로
        - 리눅스나 맥은 /를 디렉토리 구분기호로 사용하고 윈도우는 \를 사용
        - 리눅스나 맥은 /가 최상위 디렉토리이고 윈도우는 드라이브 이름이 최상위 디렉토리
    - 상대 경로
        - 현재 위치로부터의 경로
        - /를 이용해서 디렉토리를 구분
        - ./는 현재 디렉토리를 나타냄
        - ../는 상위 디렉토리를 나타냄

try:
    f = open("/Users/itaekyung/공부/현대오토에버 클라우드 4기/공부자료/log.txt", "w")
    f.write("Hello Python")
    f.write("파이선을 이용해서 파일에 기록을 합니다.")

    lines = ["Python", "Go", "Rust"]
    # f.writelines(lines)
    f.write("\n".join(lines))

except:
    print("파일 경로가 잘못되었습니다")
else:
    f.close()
