### -> init 과 del
1) `__init__`
    - 인스턴스를 생성하기 위해 호출하는 메서드 (생성자)
    - 초기화를 의미하는 initialize의 약자
    - 만들지 않으면 매개변수가 self만 있는 메서드가 자동으로 생성됨

    - 매개변수를 추가해서 생성하는 것이 가능한데 추가해서 생성하면 인스턴스를 만들 때 매개변수를 대입해 주어야 하므로 매개변수를 추가할 때는 기본값으로 설정하는 경우가 많음
2) `__del__`

    - self 이외의 매개변수가 없음
    - 외부와의 연결을 끊는데 사용
    - 외부 네트워크와의 연결이나 내부 파일 등 과의 연결 해제에 사용

### -> Inheritance(상속)
1) 개요
- 상속이란 하위 클래스가 사우이 클래스의 모든 것을 물려 받는 것
- 상위 클래스를 Based Class, Super Class 라고 하고 하위 클래스를 Derived Class, Sub Class 라고 한다.
- 하나의 클래스로부터 상속받는 것을 단일 상속이라고 하고 여러 클래스로부터 상속 받는 것을 다중 상속이라고 한다.
2) 상속의 목적
- 여러 클래스에 공통으로 존재하는 코드를 사우이 클래스에 묶어서 중복을 제거하기 위해서
- 제공되는 클래스의 기능이 부족해서 기능을 확장하거나 추가하기 위해서

3) 상속의 방법
```
class 클래스 이름(상위 클래스 이름):
```
```
class Person:
    def superMethod(self):
        print("상위 클래스의 메서드")

# Person 클래스가 상위 클래스가 되도록 설정
class Student(Person):
    def subMethod(self):
        print("하위 클래스의 메서드")

def main():
    s = Student()
    s.subMethod()
    s.superMethod()

main()
```

### -> 상위 클래스에서 만든 인스턴스 변수 호출
- 상위 클래스에서 만든 인스턴스 변수를 하위 클래스의 인스턴스가 사용하고자 할 때는 `__init__`함수를 만들어서 상위 클래스의 `__init__`를 호출해야 한다
- 호출할 때는 `super().__init__()`로 호출한다.

```
class Person:
    def __init__(self, num):
        self.num = num

    def superMethod(self):
        print("상위 클래스의 메서드")

# Person 클래스가 상위 클래스가 되도록 설정
class Student(Person):
    # 상위 클래스에 매개변수가 있는 __init__이 있는 경우는 직접 호출해줘야 함.
    def __init__(self):
        super().__init__(3)

    def subMethod(self):
        print("하위 클래스의 메서드")

def main():
    s = Student()
    s.subMethod()
    s.superMethod() # 상위 클래스의 메서드 호출
    print(s.num) # 상위 클래스의 __init__에 매개변수가 있고 초기값이 없으면 에러
    # 이런 경우에는 하위 클래스에 __init__을 만들어서 super().__init__()를 호출해줘야 함

main()
```

### -> Method Overloading (메서드 재정의)
- 상위 클래스에 존재하는 메서드와 동일한 모양의 메서드를 하위 클래스에 만드는 것
- 동일한 메서드가 존재하는 경우에는 하위 클래서의 메서드가 호출됨
- 기능 확장이 목적
- 기능 확장이 목적이므로 상위 클래서의 메서드를 호출하는 것이 일반적 (특정한 프레임워크에서는 상위 클래서의 메서드를 호출하지 않으면 에러가 발생)
- 상위 클래스의 메서드 호출 구문은 정리나 파괴를 할 때는 뒤에서 호출하고 그 이외의 경우는 먼저 호출한다.
    - 하위 클래서의 인스턴스를 생성할 때 상위 클래스의 인스턴스를 먼저 생성하고 하위 클래스의 인스턴스를 생성하기 때문

```
class Person:
    def method(self):
        print("상위 클래스의 메서드")

class Student(Person):
    # 메서드 오버라이딩: 기능 확장이 목적이므로 상위 클래스의 메서드를 호출해야 한다.
    def method(self):
        super().method()
        print("하위 클래스의 메서드")

def main():
    s = Student()
    s.method()

main()
```

### -> 다중 상속
- 여러 개의 클래스로부터 상속 받는 것
- 파이선은 다중 상속을 지원
- 상속받는 부분에 여러 개의 클래스를 나열하면 여러 클래스로부터 상속을 받음
- 여러 상위 클래스에 동일한 메서드나 속성이 존재하는 경우는 왼쪽의 클래스부터 탐색을 해서 수행

### -> 추상 (abstract)
- Abstract Method: 내용이 없는 Method로 하위 클래스에서 구현해서 사용하는 메서드로 Abstract class에 존재햐야함
    - 만들 때는 @abc.abstractmethod를 추가하면 되고 내용은 pass
- Abstract class
    - 인스턴스를 만들 수 없는 클래스
    - 상속을 통해서만 사용하는 클래스
    - 클래스 () 안에 meta = ABCMeta를 추가하면 됨.
    - 이 클래스에는 반드시 추상 메서드가 존재해야 하고 이 클래스를 상속 받은 클래스는 추상 메서드를 반드시 오버라이딩 해야함.
- 사용하는 이유
    - Polymorphism 을 구현하기 위해서
    - Template Method Pattern을 구현하기 위해서 - interface, protocol 이라고 부르는 경우가 많다.

```

import abc
class Sample(metaclass = abc.ABCMeta):
    @abc.abstractmethod
    def method(self):
        pass

# 추상클래스를 상속 받으면 반드시 추상 메서드를 구현해야 함
class Sub(Sample):
    def method(self):
        print("추상 메서드 구현")

# obj = Sample() # 추상 클래스는 인스턴스를 만들 수 없기 때문에 에러
obj = Sub()
obj.method()
```

### Template Method Pattern
- 틀을 먼저 만들고 구현
- 틀을 interface나 protocol 이라고 함
    - ex) 식당의 메뉴판을 보고 음식을 시킴

### Polymorphism 구현
```
from abc import *
class Starcraft(metaclass = ABCMeta):
    @abstractmethod
    def attack(self):
        pass

class Terran(Starcraft):
    def attack(self):
        print("테란의 공격")

class Zerg(Starcraft):
    def attack(self):
        print("저그의 공격")

class Protoss(Starcraft):
    def attack(self):
        print("프로토스의 공격")

def main():
    menu = int(input("종족을 선택하세요: (1. 테란 2. 저그 3. 프로토스) "))
    s = None
    if (menu == 1):
        s = Terran()
    elif (menu == 2):
        s = Zerg()
    elif (menu == 3):
        s = Protoss()

    s.attack()

main()
```

### Coroutine
- cooperative routine의 의미로 서로 협력하는 루틴
- 서로 대등한 관계로 특정 시점에 상대방의 코드를 실행
- 함수는 다른 함수를 호출하면 현재 함수의 제어권을 저장한 상태에서 다른 함수로 이동하고 다른 함수가 종료되면 호출한 곳으로 돌아옴
- coroutine은 종료되지 않고 나중에 다시 가서 작업을 수행할 수 있다.
- 병렬은 아니지만 왔다갔다 하면서 일을 할 수 있다.

> map, filter 함수는 병렬로 진행된다.
