# 데이터베이스
## 데이터베이스
### 데이터와 정보
- Data: 어떤 필요에 의해 수집된 자료, 특정 목적을 위해 가공하거나 평가하지 않은 값이나 사실
- Information: 어떤 목적을 위해 가공하거나 가치를 추가하거나 해서 새로운 의미를 부여한 자료
- Knowledge: 정보 패터니을 이해하여 이를 토대로 예측한 결과물, 유의미한 정보로 분류
- Wisdom: 지식을 기반으로 도출되는 창의적 아이디어

### 효율적인 데이터 관리를 위한 조건
- 데이터를 통합하여 관리
- 일관된 방법으로 관리
- 데이터 누락 및 중복 제거
- 여러 사용자가 공동으로 실시간 사용 가능

### DataBase
- 컴퓨터의 기억 능력을 활용해서 자료를 가공 및 저장, 활용하는 일체의 기능
- 자료의 집합 또는 자료를 관리하는 기술
- 통합, 저장, 운용 가능한 공용 데이터
- 사용 목적
    - 데이터와 응용 프로그램 간의 종속성을 최소화
    - 데이터의 중복으로 인한 비일관성 최소화

### 현대적인 데이터베이스가 갖추어야 할 조건
- 대용량
- 효율성: 최대한 활용하기 위한 성질 - 정교하게 설계
- 무결성(integrity): 데이터를 오류없이 사용하기 위한 성질
- 공유성
- 보안성

### DBMS
- DataBase에 대한 사용자의 모든 요구를 수행할 수 있는 기능을 갖도록 하는 각 단계별 구조와 이들 사이의 인터페이스 및 DataBase 언어로 구성된 소프트웨어
- 필요성
    - 생산성 향상
    - 기능성 향상
    - 신뢰성 향샹: 확장성(scalability)과 부하 분산(Load Balancing)을 구현

- 기능
    - 정의: 데이터의 구조를 생성하고 수정하고 삭제하는 기능(create, alter, drop, truncate, rename)
    - 조작: 실제 데이터를 삽입, 삭제, 갱신하거나 조회하는 기능 (insert, update, delete, select)
    - 제어: 정확성과 안정성을 유지하는 기능(무결성 유지, 보안, 병행 수행 제어, 복구 - Grant, Revoke, Commit, Rollback)

- 역사 - (그렇게 중요하진 않음)
    - 파일 처리(SAM, ISAM)
    - Network DB, Hierarchical-DB
    - RDBMS
    - 객체지향형, 객체관계형
    - 분산 파일 시스템, NoSQL

### RDBMS
- 개요
    - 데이터를 테이블의 집합으로 설명하는 데이터베이스 관리 시스템
- 종류
    - 상용: Oracle, MySQL, DB2, MS-SQL Server, Sybase, SAP의 HANA DB, Tibero 등
    - 오픈 소스: Maria DB(MySQL의 fork), PostgreSQL, SQLite 등
- 구성
    - Table(Relation)
    - Attribute(속성): 하나의 열, Column
    - Tuple: 하나의 행, Record나 Row
    - Domain: 속성이 가질 수 있는 값의 집합
    - Cardinality: 튜플의 수
    - Degree: Attribute의 수
    - Constraint(제약조건): 데이터의 무결성을 유지하기 위해 가져야 하는 조건
- Key
    - 구별할 수 있는 속성 또는 속성의 집합
    - Super Key: 속성의 개수에 상관없이 구별할 수 있는 속성 또는 속성의 집합
    - Candidate Key: 속성의 개수를 *최소*로 해서 구별할 수 있는 속성 또는 속성의 집합
    - Primary Key: Candidate Key 중에서 하나를 설정한 것으로 인덱스가 설정됨
    - Alternate Key: Primary Key가 아닌 Candidate Key
    - Foreign Key: 다른 테이블에서 Primary Key 인 속성(Primary Key가 아니더라도 Unique 하면 설정이 가능)

- 개체
    - Table
    - View: 자주 사용하는 SELECT 구문을 메모리에 저장하고 테이블처럼 사용하는 개체
    - Index: 데이터를 빠르게 찾을 수 있도록 해주는 개체
    - Procedure: 데이터베이스에 자주 사용하는 코드를 저장해 놓은 코드 블럭
    - Function: 입력을 받아서 데이터를 리턴하는 코드 블럭
    - Trigger: DML 문장 수행 전이나 후에 동작하는 코드 블럭
    - Sequence: 일련번호
    - Synonym: 개체에 대한 별명

- SQL(Structured Query Language)
    - RDBMS에게 질의를 하는 비절차적 언어(작성한 순서대로 동작하지 않음)
    - 종류
        - DDL: 정의어 - CREATE, ALTER, DROP, TRUNCATE, RENAME
        - DML: 조작어 - INSERT, UPDATE, DELETE
        - DCL: 제어어 - GRANT, REVOKE
        <br>
        - DQL: 질의어 - SELECT
        - TCL: 트랜잭션 제어 - COMMIT, ROLLBACK
    - 작성법
        - 한 번에 실행되는 문장의 끝은 ;
        - 문자열은 작은 따옴표 안에 입력
        - SQL 예약어는 대소문자 구분하지 않음
### NoSQL
- 개요
    - Not Only SQL의 약자로 RDBMS가 갖는 특징에 다른 특성들을 부가적으로 지원
    - 기존의 RDBMS는 트랜잭션 처리를 위해서 ACID(Atomicity, Consistency, Integrity, Durability) 특성을 제공하는데 반해 이보다는 확장성이나 성능을 향상시킴
    - 최근에는 MySQL과 같은 RDBMS에서도 NoSQL 형식의 인터페이스를 제공하고 NoSQL에서도 JOIN과 유사한 형태의 기능을 제공
- 종류
    - Key Value DB
        - Amazon의 Dynamo Paper, Riak(Full Text Search에 특화), Voldemont(Linked-In에서 사용), Redis(In Memory Database)
        - 타임라인을 표시하는 웹 애플리케이션이나 배열 형식 데이터에 빈번한 추가/삭제가 일어나는 경우 유용
    - Document DB
        - 하나의 데이터를 문서로 저장하는 구조
        - Mongo DB, CoughDb
        - 확장성이 뛰어남
        - 로깅 시스템, 인터넷 상거래 시스템, SNS 서비스 등에 활용
    - Wide Columnar Store
        - 구글의 Big Table에서 유래
        
    - Graph DB
        - Neo4J, OrientDB, SAP의 HANA(그래프 데이터베이스에 인메모리와 컴럼화 기술을 도입)
        - SNS에 많이 사용
    - Column Oriented DB
        - 행 기반 RDBMS를 사용하는 경우 분석 환경에서 한계점에 도달
        - 데이터 분석에 최적화
    - TSDB(Time-Series Database - 시계열 데이터베이스)
        - 시간을 기준으로 연속적으로 생성되는 데이터를 조회하고 저장하는데 최적화된 데이터베이스
        - 용도
            - 서버 모니터링: CPU, Memory, Disk 사용량
            - k8s Metric: Pod 수, Node 상태, 네트워크 트래픽
            - IoT 센서: 온도, 습도, 진동
            - 주식이나 환율 데이터
            - 스마트 팩토리 설비 데이터
            - 애플리케이션 로그 및 성능 지표
            - 자동차 운행 정보

### Maria DB
- 개요
    - SQL에 기반을 둔 관계형 데이터베이스로 Open Source로 제공
    - 작업 단위
        - 데이터베이스 > 테이블
        - 데이터베이스는 사용자와 상관없이 생성되며 사용자에게 권한을 부여해서 사용
- 설치
    - MariaDB 서버 설치
    - 접속도구 설치: DBeaver 설치

- 외부 접속 허용
    - 설정 파일에서 bind-access = 0.0.0.0으로 수정해주면 됨.
    - 설정 파일 위치
        - windows: C:\Program Files\Maria DB 12\data\my.ini
        - Mac에서 homebrew로 설치한 경우: /opt/homebrew/etc/my.cnf
        - ubuntu linux: /etc/mysql/mariadb/conf.d/50-server.cnf
- 유저 생성
    - `create user '계정'@'접속위치' identified by '비밀번호';`
        - 접속 위치를 작성할 때 %를 기재하면 아무 곳에서나 접속이 가능
        - 127.0.0.1 이나 localhost를 기재하면 현재 컴퓨터에서만 접속 가능
    - `GRANT ALL PRIVILEGES ON 사용할 데이터베이스 TO '계정'@'접속위치';`
        - 사용할 데이터베이스 이름 대신에 `*.*` 을 하면 모든 데이터베이스를 사용 가능
```
  CREATE USER 'username'@'%' IDENTIFIED BY 'password';
  GRANT ALL PRIVILEGES ON testdb.* TO 'username'@'%';
  FLUSH PRIVILEGES;
```

- 데이터베이스 관련 명령
    - 데이터베이스 목록 확인
        - `show databases;`
    - 현재 사용 중인 데이터베이스 확인
        - `select database();`
    - 데이터베이스 생성
        - `create database 이름;`
    - 데이터베이스 사용 설정
        - `use 이름;`
    - 데이터베이스 삭제
        - `drop database 이름;`

- SELECT(조회)의 기본형식
```
SELECT [DISTINCT] {*, 컬럼 [별명],...}
FROM 테이블
[WHERE 조건]
[GROUP BY 그룹화 할 조건]
[HAVING GROUP BY 이후의 조건]
[ORDER BY 정렬을 수행할 조건]
[LIMIT 행의 개수[OFFSET 시작위치]]
```
- 실행 순서
    - FROM -> WHERE -> GROUP BY -> HAVING -> SELECT -> ORDER BY -> LIMIT

- 테이블의 모든 데이터 확인
    `SELECT * FROM 테이블이름;`

- 특정 컬럼 조회
    ```
    SELECT name, popu
    FROM tCity;
    ```

- 컬럼 이름에 별명 사용
    - SELECT 절에 컬럼 이름을 기술할 때 컬럼이름 [AS] 별명의 형태로 설정
    - 별명에 공백이 있거나 대문자 또는 한글 등이 있으면 ""로 묶어준다.
    - AS는 생략 가능
    - 별명은 ORDER BY 절에서만 사용 가능

- 계산 식의 출력
    - SELECT 절에 계산식 사용 가능
    - 계산식을 사용할 때는 별명을 이용하는 것이 좋다.
    - 계산식은 계산식 자체가 컬럼 이름이 돼서 출력됨
    - Application에서 게산식을 가져온 경우 컬럼 이름이 제대로 만들어지지 않는 경우가 있다.
    - tCity 테이블에서 name과 popu에 10000을 곱한 값을 조회
        ```
        SELECT name, popu * 10000 AS "인구"
        FROM tCity;
        ```
    - 컬럼이 아닌 단순 계산식이ㅡ 경우는 FROM을 생략하는 것이 가능

- CONCAT
    - 여러 개의 컬럼을 묶어서 하나의 문자열을 만들어 주는 함수
    - EMP 테이블에서 ENAME, JOB을 하나로 묶어서 조회
        ```
        SELECT CONCAT(ENAME,' ', JOB) AS "Employees"
        FROM EMP;
        ```

- DISTINCT
    - SELECT 절에 맨 앞에 한 번만 나올 수 있음
    - 컬럼의 중복을 제거해주는 옵션
    - 컬럼 이름이 2개 이상이면 모든 컬럼의 값이 중복되는 경우에 제거
    - tCity 테이블에서 region 컬럼의 값을 중복을 제거하고 조회
        ```
        SELECT DISTINCT region
        FROM tCity;
        ```
    - 2개 이상의 컬럼 이름 나열 - region과 name 모두 일치하는 데이터만 제거
        ```
        SELECT DISTINCT region, name
        FROM tCity;
        ```
    
- ORDER BY - 정렬
    - ORDER BY 절이 없으면 Maria DB는 기본키 값을 기준으로 오름차순 정렬해서 데이터를 조회
    - ORDER BY 절을 이용하면 원하는 대로 정렬 가능
    - 정렬 방법
        - ORDER BY 칼럼이름 [ASC|DESC]
            - ASC가 기본으로 생략가능한데 오름차순 정렬이고, DESC는 내림차순 정렬
            - SELECT에서 만든 별명 사용 가능
            - 계산식 사용 가능
            - 컬럼 이름 대신에 컬럼의 인덱스도 가능
            - 여러 개의 정렬 조건을 설정하면 앞의 것을 기준으로 정렬을 하고 값이 같은 경우 뒤의 조건을 적용
    - NULL 값은 오름차순에서는 제일 나중에, 내림차순에서는 제일 먼저 조회

    - 데이터는 정렬해서 조회하는 것을 권장
    - tCity 테이블의 모든 데이터를 조회하는데 popu의 오름차순으로 조회
        ```
        SELECT *
        FROM tCity
        ORDER BY popu ASC;
        ```
    - popu 컬럼이 3번째 있으므로 이름 대신에 3이라고 입력해도 됨
        ```
        SELECT *
        FROM tCity
        ORDER BY 3 DESC;
        ```
    - 정렬을 할 때 별명 사용하는 것이 가능

- WHERE 절
    - 개요
        - 읽을 레코드의 조건을 기술
        - WHERE 절이 생략되면 모든 레코드를 전부 조회
    - 비교 연산자
        - =
        - >
        - <
        - >=
        - <=
        - <>, !=, ^=, NOT 컬럼이름 = : 다른경우
        - <=> : 2개의 값이 같으면 1, 그렇지 않으면 0
    > MySQL과 MariaDB는 저장을 할 때는 대소문자를 구분하지만 WHERE 절에서는 기본적으로 대소문자 구분을 하지 않음
    
    - tCity 테이블에서 metro 값이 y인 데이터의 모든 컬럼을 조회 (대소문자 구분 안함)
        ```
        SELECT * 
        FROM tCity
        WHERE metro = 'y';
        ```
    - 대소문자 구분
        - 컬럼 이름 앞에 BINARY를 추가해서 해결
            ```
            SELECT *
            FROM tCity
            WHERE BINARY metro = 'Y';
            ```
        - 문자열의 인코딩 방식을 설정할 때 콜레이션을 설정
            - MariaDB에서는 기본적으로 인코딩을 설정할 때 뒤에 _ci가 붙는 인코딩을 사용하는데 ci가 Case Insensitive의 약자로 대소문자 구분을 하지 않음
            - 테이블을 만들 때 콜레이션을 뒤에 _bin 이나 _cs가 붙는 콜레이션으로 설정해야 함
            - 테이블을 생성하는 경우
                `CREATE TABLE 테이블이름(컬럼이름 자료형 COLLATE utf8mb4_bin,)`
            - 기존에 만들어진 테이블의 경우
                `ALTER TABLE 테이블이름 MODIFY 컬럼이름 자료형 COLLATE utf8mb4_bin;`

    - 테이블의 구조 확인
        - `desc 테이블명;`
- NULL 비교
    - NULL: 아직 알려지지 않은 값
    - 데이터베이스에서는 NULL을 표현하기 위해서는 하나의 바이트를 더 가져야함.
    - NULL을 찾을 때는 IS NULL 그리고 NULL이 아닌 데이터를 찾을 때는 IS NOT NULL을 사용해야함.
    - EMP 테이블에는 COMM 컬럼이 있는데 이 컬럼에 NULL이 존재함.
    - EMP 테이블에서 COMM의 값이 NULL인 모든 데이터를 조회
        ```
        SELECT *
        FROM EMP
        WHERE COMM IS NULL;
        ```
    
- LIKE
    - 문자열의 패턴을 검색
    - 와일드 카드 문자
        - %: 문자 개수에 상관없음
        - _: 하나의 문자
    - []: 문자 리스트 중 하나와 일치
    - [^]: 문자 리스트와 일치하지 않는 것
    - 와일드 카드 문자를 검색할 때는 ESCAPE를 이용
    
    - EMP 테이블에서 ENAME에 t가 들어가는 데이터를 조회
        ```
        SELECT ENAME
        FROM EMP
        WHERE ENAME LIKE '%t%';
        ```

    - EMP 테이블에서 ENAME이 t로 시작하는 데이터를 조회
        ```
        SELECT ENAME
        FROM EMP
        WHERE ENAME LIKE 't%';
        ```
    - EMP 테이블에서 ENAME이 네번째 글자가 t인 데이터를 조회
        ```
        SELECT ENAME
        FROM EMP
        WHERE ENAME LIKE '___t%';
        ```
    - EMP 테이블에서 ENAME에 t가 2번 이상 들어가는 데이터를 조회
        ```
        SELECT ENAME
        FROM EMP
        WHERE ENAME LIKE '%t%t%';
        ```
    - EMP 테이블에서 HIREDATE가 1980년인 데이터의 ENAME과 HIRENATE를 조회
        ```
        SELECT ENAME, HIREDATE 
        FROM EMP
        WHERE HIREDATE LIKE '1980%';
        ```
    - EMP 테이블에서 ENAME에 t가 안 들어가는 데이터를 조회
        ```
        SELECT ENAME
        FROM EMP
        WHERE ENAME NOT LIKE '%t%';
        ```
    - 와일드 카드 문자를 검색할 때는 ESCAPE를 이용
        - %나 _와일드 카드 문자라서 바로 검색이 안됨
        - ESCAPE를 이용해서 문자를 하나 정의하고 이 문자 뒤에 나오는 것은 와일드 카드 문자로 해석하지 않도록 해주어야함.
        - sale이라는 컬럼에서 30%로 끝나는 데이터를 조회
            ```
            WHERE sale LIKE '%30$%' ESCAPE '$'
            ```

    - BETWEEN
        - BETWEEN A AND B 형태로 사용해서 A부터 B까지의 데이터를 조회
        - 반드시 B가 A보다 크거나 같아야 함
        - tCity 테이블에서 popu 값이 50보다 크거나 같고 100보다 작거나 같은 데이터를 조회
            ```
            SELECT *
            FROM tCity
            WHERE popu >= 50 AND popu <= 100;
            ```
            ```
            SELECT *
            FROM tCity
            WHERE popu BETWEEN 50 AND 100;
            ```
    - IN 연산자
        - 여러 개 중에 하나와 일치하면 되는 경우를 조회
        - tCity 테이블에서 region이 '경상'이거나 '전라'인 데이터를 조회
            ```
            SELECT *
            FROM tCity
            WHERE region = '경상' OR region = '전라';
            ```
            ```
            SELECT *
            FROM tCity
            WHERE region IN ('경상', '전라');
            ```
    - LIMIT
        - 행의 개수를 제한
        - 사용법
            - LIMIT [건너뛸 개수,] 조회할 개수
        - tCity 테이블에서 5개의 데이터를 조회
            ```
            SELECT * 
            FROM tCity
            LIMIT 5
            ```
        - tCity 테이블 앞의 3개의 데이터를 제외하고 5개의 데이터를 조회
            ```
            SELECT * 
            FROM tCity
            LIMIT 3, 5
            ```
            ```
            SELECT * 
            FROM tCity
            LIMIT 5 OFFSET 3
            ```
    
### 검색 결과를 파일로 만들기
- 기본 형식
    ```
    SELECT ***
    INTO OUTFILE '파일경로'
    [CHARACTER SET 인코딩방식]
    [export_options]
    ```
- export_options
    [
        {FIELDS | COLUMNS}
        [TERMINATED BY '문자']
        [[OPTIONALLY] ENCLOSED BY 'char']
        [ESCAPED BY 'char']
    ]
    [LINES
        [STARTING BY '문자']
        [TERMINATED BY '문자']
    ]

- EMP 테이블의 데이터를 파일로 내보내기
    ```
    SELECT * 
    INTO OUTFILE ''
    FIELDS TERMINATED BY ','
    ENCLOSED BY '"'
    FROM EMP
    ```
