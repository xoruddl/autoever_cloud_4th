# Redis

## Redis 사용

### 1) 설치

- public cloud 와 같은 곳의 SaaS(서비스-애플리케이션 서비스)를 이용

    - AWS ElastiCache

- 운영체제에 직접 설치
- Docker 나 k8s와 같은 컨테이너를 이용해서 설치

### 2) 설정

#### 설정파일 위치

- **Linux**: /etc/redis/redis.conf
- **Mac**: /opt/homebrew/etc/redis.conf
- **Window**: 설치된 디렉토리의 redis.windows-service.conf

#### 설정

- **bind**: redis를 사용할 수 있는 IP를 설정
    - 127.0.0.1 하면 로컬에서만 접속 가능
    - 0.0.0.0 하면 모든 곳에서 접속 가능

- 각 서비스는 제한된 ip에서만 접근 가능하게 만들어야 한다.
    - 나중에 ip가 많이 필요하므로 로드밸런서가 있어야 한다.

- **port**: 애플리케이션을 구분하기 위한 번호로 기본적으로 6379이고 내부에서 redis끼리 통신하기 위해서 16379번을 사용

- **protected-mode**: 패스워드를 설정해야만 redis에 접근할 수 있도록 하는 설정으로 기본값은 yes

- **requirepass/ masterauth**: 패스워드

- **daemonize**: 프로세스를 데몬으로 실행하고자 할 때 사용하는 것으로 기본값은 no
    - 서바나 디비는 거의 다 daemon

- **dir**: 작업 디렉토리로 기본값을 ./(현재 디렉토리)

- **tcp-backlog**: 초당 클라이언트 연결 개수로 default는 511
    - 백로그는 애플리케이션에 맞게 설정을 해야한다.
    - 작업시간이 긴 작업이면 backlog 숫자를 낮출 필요가 있다.
    - overhead: 일하는데 필요한 작업
    - context-switching = overhead

    - thrashing: 시스템(주로 OS의 가상 메모리)이 실제 작업보다 페이지 교체(paging)에 더 많은 시간을 쓰면서 성능이 급격히 떨어지는 현상

- **timeout**: 클라이언트에서 서버 접속 시의 timeout 값으로 기본은 0

- **maxclients**: 최대 접속 가능한 client 수로 기본값은 10000 (컴퓨터 수가 아닌 세션의 개수)

- **tcp-keepalive**: 클라이언트가 반응을 보이지 않을 때 서버가 접속을 제거하는 시간으로 기본값은 300

#### 설정을 변경하면 Server를 다시 시작해야 적용된다. 

### 3) 접속

#### redis를 설치하면 redis-cli 라고 하는 접속 도구를 제공

- 바로 실행하면 로컬에 있는 redis 서버에 접속

#### 원격 접속

- redis-cli -h <ip주소> -p <포트번호> -a 패스워드
- ip 주소를 생략하면 127.0.0.1 이고 포트를 생략하면 6379

### 4) Redis의 자료구조

- String
- Hash
- List: Linked List
- Set
- Sorted Set
- Bitmap: 비트 연산을 사용할 수 있는 자료구조
- Hyperloglog: 집합의 아이템 개수를 추정할 수 있는 알고리즘이자 자료구조
- Geospatial Index: 지점과 위도와 경도를 사용할 수 있는 자료구조로 위도와 경도를 계산해서 두 지점 간의 거리를 구할 수 있는 명령어 제공
- Stream: 이벤트 로그를 저장하는 것으로 이를 이용해서 메시지 서비스를 제공

#### String

- 가장 간단하게 데이터를 저장할 수 있는 구조
- 512MB까지 저장 가능
- 키와 데이터가 직접 연결되는 유이란 자료구조
- 저장
    - `SET 키 데이터`
- 읽기
    - `GET 키`
- NX 옵션과 XX 옵션을 추가해서 키가 없을 때만 저장하거나 키가 있는 경우에만 저장하도록 하는 것도 가능
    ```log
    127.0.0.1:6379> SET hello world
    OK
    127.0.0.1:6379> get hello
    "world"
    127.0.0.1:6379> SET hello newvalue NX
    (nil)
    127.0.0.1:6379> SET hello newvalue XX
    OK
    127.0.0.1:6379> GET hello
    "newvalue"
    ```
- 숫자도 저장가능하고 숫자의 경우는 증감 연산 가능
    ```
    SET cnt 0
    INCR cnt
    INCRBY cnt 50
    ```
    ```log
    127.0.0.1:6379> SET cnt 0
    OK
    127.0.0.1:6379> INCR cnt
    (integer) 1
    127.0.0.1:6379> GET cnt
    "1"
    127.0.0.1:6379> INCRBY cnt 50
    (integer) 51
    127.0.0.1:6379> GET cnt
    "51"
    ```

- 여러 개를 저장하고자 할 때는 MSET과 MGET 사용
    ```log
    127.0.0.1:6379> MSET name adam age 50 weight 77
    OK
    127.0.0.1:6379> MGET name age weight
    1) "adam"
    2) "50"
    3) "77"
    ```

#### List

- 하나의 키에 여러 개의 데이터를 저장하는 자료구조
- 하나의 list에는 43억개의 데이터를 저장할 수 있음
- 인덱스를 이용해서 데이터에 접근하는 것이 가능
- 큐와 스택의 용도로 사용 가능
- 맨 앞과 맨 뒤에서 데이터를 삽입하고 꺼내는 것도 가능
- 삽입: `LPUSH`, `RPUSH`

    ```log
    127.0.0.1:6379> LPUSH mylist adam
    (integer) 1
    127.0.0.1:6379> RPUSH mylist hunt
    (integer) 2
    127.0.0.1:6379> LPUSH mylist jessica julia
    (integer) 4
    ```

- 범위 내 데이터 조회: `LRANGE key start stop`
    - 맨 끝에서부터 조회할 때는 -1, -2 형태의 음수를 사용

        ```log
        LRANGE mylist 0 -1
        ```

- 맨 끝의 데이터를 삭제하면서 조회: `LPOP`, `RPOP`
    - 횟수도 설정 가능

    ```log
    127.0.0.1:6379> LRANGE mylist 0 -1
    1) "julia"
    2) "jessica"
    3) "adam"
    4) "hunt"
    127.0.0.1:6379> LPOP mylist
    "julia"
    127.0.0.1:6379> LRANGE mylist 0 -1
    1) "jessica"
    2) "adam"
    3) "hunt"
    ```

- 범위를 제외하고 전부 삭제: `LTRIM`으로 사용법은 LRANGE와 동일

    - LTRIM을 적절히 활용하면 고정된 길이의 큐를 쉽게 유지할 수 있음
    - 특정 개수의 로그 데이터를 유지하고자 하는 경우
    ```log
    LPUSH logdata <데이터>
    LTRIM 0 99
    # 항상 백개의 최신 로그가 유지됨.
    ```

- 중간에 데이터 삽입 가능: `LINSRT`
    
    - `LINSERT 리스트이름 BEFORE 나 AFTER 기준값 추가할 값`

    ```log
    127.0.0.1:6379> LRANGE mylist 0 -1
    1) "jessica"
    2) "adam"
    3) "hunt"
    127.0.0.1:6379> LINSERT mylist BEFORE adam mark
    (integer) 4
    127.0.0.1:6379> LRANGE mylist 0 -1
    1) "jessica"
    2) "mark"
    3) "adam"
    4) "hunt"
    ```

- 데이터 수정: LSET
    - `LSET 리스트이름 키 인덱스값`

    ```log
    127.0.0.1:6379> LRANGE mylist 0 -1
    1) "jessica"
    2) "mark"
    3) "adam"
    4) "hunt"
    127.0.0.1:6379> LSET mylist 2 adamm
    OK
    127.0.0.1:6379> LRANGE mylist 0 -1
    1) "jessica"
    2) "mark"
    3) "adamm"
    4) "hunt"
    ```

- 특정 인덱스의 데이터 조회: `LINDEX`

    ```log
    127.0.0.1:6379> LRANGE mylist 0 -1
    1) "jessica"
    2) "mark"
    3) "adamm"
    4) "hunt"
    127.0.0.1:6379> LINDEX mylist 2
    "adamm"
    ```

- 특정 데이터 삭제: `LREM`

    - `LREM key 횟수 값`

    ```log
    127.0.0.1:6379> LREM mylist 2 adamm
    (integer) 1
    127.0.0.1:6379> LRANGE mylist 0 -1
    1) "jessica"
    2) "mark"
    3) "hunt"
    127.0.0.1:6379>
    ```

- 데이터 개수: `LLEN 키`
    ```log
    127.0.0.1:6379> LLEN mylist
    (integer) 3
    ```

> redis는 배열을 저장할 때 linked list를 이용
>
> 메모리라 어차피 빨라서 linked list로 구현해도 괜찮다.


#### Set

- 중복없이 데이터를 저장한다.
- 집합 관련 기능을 제공

- 삽입: `SADD 키 데이터나열`
    ```log
    127.0.0.1:6379> SADD myset A A A B B
    (integer) 0
    ```

- 데이터조회: `SMEMBERS`
    ```log
    127.0.0.1:6379> SMEMBERS myset
    1) "A"
    2) "B"
    ```

- 특정 값이 있는지 조회: `SISMEMBER`
    ```log
    127.0.0.1:6379> SISMEMBER myset A
    (integer) 1
    127.0.0.1:6379> SISMEMBER myset B
    (integer) 1
    127.0.0.1:6379> SISMEMBER myset C
    (integer) 0
    ```

- 무작위 추출: `SRANDMEMBER`
    ```log
    127.0.0.1:6379> SRANDMEMBER myset
    "A"
    127.0.0.1:6379> SRANDMEMBER myset
    "B"
    ```

- 삭제: `SREM`
    ```log
    127.0.0.1:6379> SREM myset A
    (integer) 1
    127.0.0.1:6379> SMEMBERS myset
    1) "B"
    ```

- 마지막 데이터 삭제: `SPOP`
    ```log
    127.0.0.1:6379> SPOP myset
    "B"
    127.0.0.1:6379> SMEMBERS myset
    (empty array)
    ```

- 집합 관련 연산
    - `SINTER`
    - `SUNION`
    - `SDIFF`

    ```log
    127.0.0.1:6379> SADD myset1 A B
    (integer) 2
    127.0.0.1:6379> SADD myset2 B C
    (integer) 2
    127.0.0.1:6379> SINTER myset1 myset2
    1) "B"
    127.0.0.1:6379> SUNION myset1 myset2
    1) "C"
    2) "A"
    3) "B"
    127.0.0.1:6379> SDIFF myset1 myset2
    1) "A"
    ```

#### Hash

- 하나의 키에 여러 개의 필드와 값을 가질 수 있는 자료구조
- 필드는 해시 내에서는 유일해야 한다.
- 생성 및 필드 추가 및 수정은 `HSET`
    ```log
    127.0.0.1:6379> HSET myhash name adam
    (integer) 1
    127.0.0.1:6379> HSET myhash age 30
    (integer) 1
    ```
    ```log
    127.0.0.1:6379> HSET myhash age 29
    (integer) 0
    127.0.0.1:6379> HVALS myhash
    1) "adam"
    2) "29"
    ```

- 특정 필드의 값을 가져오는 건 `HGET`
    ```log
    127.0.0.1:6379> HGET myhash age
    "29"
    ```

- 여러 개의 필드의 값 가져오기는 `HMGET`
    ```log
    127.0.0.1:6379> HMGET myhash name age
    1) "adam"
    2) "29"
    ```

- 모든 필드 가져오기 `HGETALL`
    ```log
    127.0.0.1:6379> HGETALL myhash
    1) "name"
    2) "adam"
    3) "age"
    4) "30"
    ```

- 모든 키 가져오기 `HKEYS`
    ```log
    127.0.0.1:6379> HKEYS myhash
    1) "name"
    2) "age"
    ```

- 모든 값 가져오기 `HVALS`
    ```log
    127.0.0.1:6379> HVALS myhash
    1) "adam"
    2) "29"
    ```

- 특정 키 삭제 `HDEL`
    ```log
    127.0.0.1:6379> HDEL myhash name
    (integer) 1
    127.0.0.1:6379> HGETALL myhash
    1) "age"
    2) "29"
    ```

#### Sorted Set (ZSet)

- score 값에 따라 정렬되는 고유한 문자열의 집합
- 모든 아이템은 스코어와 값 쌍을 가지며 저장될 때부터 스코어 값으로 정렬돼 저장
- 스코어가 동일하면 데이터의 사전 순서대로 정렬
- list와 sortedset 모두 순서를 갖는 자료구조라서 인덱스를 이용해서 아이템에 접근할 수 있음
    - 인덱스를 이용한 접근 속도는 sortedset이 빠르다.
    - list는 linked list이기에 데이터의 접근 속도가 O(n)이지만 Sorted Set은 Hash를 이용해서 인덱스를 저장하기 때문에 접근 속도가 O(log n)

- 데이터 저장은 `ZADD 키 스코어 값 스코어 값...`
    ```log
    127.0.0.1:6379> ZADD myzset 100 user:B
    (integer) 1
    127.0.0.1:6379> ZADD myzset 150 user:C 150 user:A 200 user:D
    (integer) 3
    ```

- 옵션 
    - XX
    - NX
    - LT: 업데이트하고자 하는 스코어가 기존 아이템의 스코어보다 작을 떄만 업데이트하고 없으면 추가
    - GT: 업데이트하고자 하는 스코어가 기존 아이템의 스코어보다 클 떄만 업데이트하고 없으면 추가

- 아이템 조회
    - 인덱스로 조회: `ZRANGE 이름 start end WITHSCORES [REV]`
        ```log
        127.0.0.1:6379> ZRANGE myzset 0 3 WITHSCORES REV
        1) "user:D"
        2) "200"
        3) "user:C"
        4) "150"
        5) "user:A"
        6) "150"
        7) "user:B"
        8) "100"
        ```

        ```log

        ```

    - 점수로 조회: `ZRANGE 이름 start end BYSCORE WITHSCORES [REV]`
        - 이떄 BYSCORE 앞에 (를 추가하면 start 값은 제외
    
#### Bitmap

- String 자료구조에 bit 연산을 수행할 수 있도록 확장된 형태

#### Hyperloglog

- 집합의 원소 개수인 카디널리티를 추정할 수 있는 자료구조
- 대량의 데이터에서 중복되지 않는 고유한 값을 집계할 때 유용하게 사용할 수 있는 자료구조
- set과 같은 데이터 구조에서는 중복을 피하기 위해 저장된 데이터를 전부 기억하고 있기 때문에 저장되는 데이터가 많아지면 메모리를 많이 사용
- 입력되는 데이터 자체를 저장하지 않고 자체적인 방법으로 데이터를 변경해서 처리하기 때문에 데이터 개수에 상관없이 일정한 메모리를 유지하면서 중복되지 않는 유일한 원소의 개수를 계산할 수 있음
- 오차는 어느 정도 발생

- 추가할 때는 `PFADD` 를 이용하고 데이터 개수는 `PFCOUNT`
    ```log
    127.0.0.1:6379> PFADD members 100
    (integer) 1
    127.0.0.1:6379> PFADD members 100
    (integer) 0
    127.0.0.1:6379> PFADD members 150
    (integer) 1
    127.0.0.1:6379> PFADD members 200
    (integer) 1
    127.0.0.1:6379> PFADD members 300
    (integer) 1

    127.0.0.1:6379> PFCOUNT members
    (integer) 4
    ```

#### Geospatial

- 데이터를 위도와 경도 쌍의 집합으로 저장하는 자료구조
- 내부적으로는 sorted set
- 삽입은 `GEOADD`를 이용하고, 데이터 조회는 `GEOPOS`, 데이터 사아의 거리는 `GEODIST`
    - `GEOADD`
    ```log
    127.0.0.1:6379> GEOADD travel 14.3996 50.0992 prague
    (integer) 1
    ```

    - `GEOPOS`
    ```log
    127.0.0.1:6379> GEOADD travel 127.0016 37.5642 seoul -122.4345 37.7853 SanFrancisco
    (integer) 2
    127.0.0.1:6379> GEOPOS travel seoul
    1) 1) "127.00160175561905"
       2) "37.56419877947457"
    ```

    - 거리 계산
    - `GEODIST`
    ```log
    127.0.0.1:6379> GEODIST travel seoul prague
    "8253000.2448"
    ```

- 특정 위치를 기준으로 원하는 거리 내의 아이템 검색
    - GEOSEARCH travel 이름 거리 옵션

    - BYRADIUS 옵션을 이용하면 반경이고 BYBOX는 직사각형 거리

### 4) 키 관리

- 키는 존재하지 않을 때 아이템을 넣으면 자동으로 생성
- 모든 아이템을 삭제하면 키도 자동으로 삭제
- 키 삭제: `DEL 키`
- 키의 존재여부: `EXISTS 키`
- 특정 패턴 이용해서 키의 존재 여부 확인: `KEYS pattern(정규표현식)`
- 이름 변경: `RENAME key 새로운이름`
- 복제: `COPY 기존키 새로운키`
- 모든 키 삭제: `FLUSHALL [ASYNC | SYNC]`
- UNLINK 키: 키의 데이터를 백그라운드에서 다른 스레드에 의해 처리
- 데이터의 유효 기간 설정
    - `EXPIRE key 유지시간으로초단위 [NX | XX | GT | NT]`
    - `EXPIREAT key 타임스탬프 [NX | XX | GT | NT]`

# Python 과 데이터베이스 연결

## 1. Python DB API

### 개요

- 파이선에서 데이터베이스를 액세스하기 위해 만든 API
- 표준 API는 데이터베이스를 연결하고 쿼리를 수행하고 연결을 닫는 기본적인 DB작업과 관련된 기능들을 정의
- 이러한 표준 API를 응용해서 액세스 하는 방식으로 ORM(Object Relational Mapping)이 있는데 이 방식은 RDBMS의 테이블과 객체를 매핑시켜서 쿼리를 직접 실행하지 않고 객체의 메서드를 호출해서 데이터베이스를 사용하도록 하는 방식

- 데이터베이스 접속에 필요한 정보
    - URL: 필수
    - PORT: 필수 정보인데 데이터베이스 종류에 따라서 기본 포트를 사용하는 경우에는 생략이 가능

    - 데이터베이스이름
    - 계정
    - 비밀번호

## 2. MongoDB 연동

### 1) 과정

- pymmongo 패키지 설치 - 드라이버
- 데이터베이스 연결
    - `변수 = pymongo.MongoClient("URL", 포트번호)`

- 데이터베이스 생성 또는 연결
    - `데이터베이스변수 = 변수.데이터베이스이름`

- 컬렉션 연결
    - `컬렉션변수 = 데이터베이스변수.컬렉션`

- 데이터베이스 작업 수행

- 연결 해제
    - `변수.close()`

### 2) 데이터베이스 연결

- 패키지 설치: `pip install pymongo`
    - mac에서 brew로 설치하면 가상환경에서 해야 동작한다. guide.md 참고

- 데이터베이스 연결 및 해제

### 3) 데이터 삽입

- `insert_one` 메서드나 `insert_many` 메서드 이용

```python
# 디비 연결 실습: mongoDB

from pymongo import MongoClient

try:
    # 데이터베이스 연결
    # 실제 접속은 안함
    conn = MongoClient('127.0.0.1')

    # 디비 설정: 없으면 생성
    db = conn.mymongo
    print("연결 성공")

    users = db.users

    # 데이터 한 건 삽입
    users.insert_one({'empno': '10001', 'name': 'kim'})

    # 데이터 여러 건 삽입
    users.insert_many([
        {'empno': '10002', 'name': 'lee'},
        {'empno': '10003', 'name': 'park'}
    ])
    print("데이터 삽입 성공")

except Exception as e:
    print("연결 실패", e)

else:
    conn.close()
```

### 4) 데이터 조회

- `find` 와 `find_one` 함수 이용
    - 결과를 딕셔너리로 리턴
    - 여러 개의 데이터이면 딕셔너리의 커서이다.
    - for를 돌려서 해석을 해야한다.

```python
# 디비 연결 실습: mongoDB

from pymongo import MongoClient

try:
    # 데이터베이스 연결
    # 실제 접속은 안함
    conn = MongoClient('127.0.0.1')

    # 디비 설정: 없으면 생성
    db = conn.mymongo
    print("연결 성공")

    users = db.users

    # 데이터를 읽어왔는데 전체 데이터를 출력하려고 하면 문자열이 출력
    results = users.find()
    # print(results)

    for result in results:
        print(result)

except Exception as e:
    print("연결 실패", e)

else:
    conn.close()
```

### 5) 데이터 수정

- `update_one`
- `update_many`

```python
# 디비 연결 실습: mongoDB

from pymongo import MongoClient

try:
    # 데이터베이스 연결
    # 실제 접속은 안함
    conn = MongoClient('127.0.0.1')

    # 디비 설정: 없으면 생성
    db = conn.mymongo
    print("연결 성공")

    users = db.users

    users.update_many(
        {'empno': '10001'},
        {"$set": {"name": "김좌진"}}
    )

    results = users.find()

    for result in results:
        print(result['name'])

except Exception as e:
    print("연결 실패", e)

else:
    conn.close()
```

## 3. Redis 사용

### 1) Redis

- 패키지: `redis`

- 연결
    - `redis.StrictRedis(host = "IP", port = 포트번호, ..)`

```python
# redis 연결 실습

import redis

with redis.StrictRedis(host = "127.0.0.1", port = 6379) as con:
    print(con)
```

- 문자열 사용

```python
import redis

redis_pool = redis.ConnectionPool(host = "127.0.0.1", port = 6379,
                                  max_connections = 4)

with redis.StrictRedis(connection_pool = redis_pool) as con:
    # print(con)
    
    # 문자열 저장
    con.set("album", "genesis")
    # 읽어오기
    print(con.get("album"))
```

- 키의 만료 시간 설정

```python
import redis

redis_pool = redis.ConnectionPool(host = "127.0.0.1", port = 6379,
                                  max_connections = 4)

with redis.StrictRedis(connection_pool = redis_pool) as con:
    
    # 이 데이터의 만료 시간은 30초
    con.set("album", "genesis", 30)
    # 확인
    print(con.ttl("album"))
    # 만료시간 수정
    con.expire("album", 50)
    print(con.ttl("album"))
    print(con.get("album"))
```

## 4. MariaDB 연동

### 1) 연결

- 패키지: pymysql

```python
pymysql.connect(host = "디비 접속 위치", port = 포트번호, user = "게정", password = "비밀번호",
                db = "디비이름", charset = "인코딩방식")
```

### 2) 사용

```
커서  = 연결객체.cursor()
cursor.execute(SQL)
cursor.commit()
```

### 3) 연결 해제
- `연결객체.close()`

### 4) 조회의 경우

- execute를 실행하고 `cursor.fetchall()`을 호출하면 읽은 전체 데이터를 튜플의 튜플로 만들어 준다.

### 5) 주의할 점

- python 최신 버전에서는 pymysql이 설치가 제대로 안되는 경우가 있으므로 mysql(mariadb)을 사용할 때는 `python 3.14` 보다는 `python 3.13` 을 권장

- 실제 데이터베이스 연동은 특별한 경우가 아니면 ORM을 사용
    - django의 경우는 내장된 ORM을 사용하지 않고 fast API 같은 경우는 별도로 ORM을 설치해서 사용
