# MongoDB

## CRUD

### 1) 데이터 삽입 (도큐먼트 생성)

#### insert 의 두번쨰 매개변수로 ordered 옵션을 설정할 수 있는데 이 옵션에 true가 기본값인데 true가 설정되면 싱글 스레드로 동작하고 false로 설정하면 멀티 스레드를 이용해서 삽입한다.

- sample 컬렉션을 생성하는데 name의 값을 중복 없이 저장하도록 생성
    - `db.sample.createIndex({name: 1}, {unique: true})`

- 데이터 삽입
    - `db.sample.insert({name: "park"})`

- 여러 개의 데이터를 동시에 삽입
    - `db.sample.insert([{name: "kim"}, {name: "park"}, {name: "lee"}, {name: "choi"}])`

    - park이 중복돼서 에러가 발생
    - 지금은 싱글 스레드 형태로 삽입하므로 park이 에러가 나면 뒤의 2개의 데이터는 삽입되지 않는다.
    - `db.sample.find()`

- sample 컬렉션 삭제
    `db.sample.drop()`

- 멀티 스레드로 삽입
    - `db.sample.insert([{name: "kim"}, {name: "park"}, {name: "lee"}, {name: "choi"}], {ordered: false})`
    - park이 중복돼서 에러는 발생해도 나머지 데이터는 삽입됨

#### ObjectId

- MongoDB에서는 데이터를 삽입할 때 _id 항목의 값을 설정하지 않으면 Mongo DB가 ObjectId라는 자료형을 일련번호를 만들어서 자동 삽입
- 12byte로 구성되어 있고 new ObjectId()를 이용해서 직접 생성 가능

    ```js
    var newId = new ObjectId()
    db.sample.insert({_id: newId, name: "user01"})
    ```

#### insertOne과 insertMany
- insert 함수는 deprecated
- 하나의 데이터를 삽입할 때는 insertOne 그리고 여러 개의 데이터를 삽입할 때는 insertMany를 사용하도록 권장
    - `db.sample.insertOne({name: "karroid"}, {password: "1111"})`

    - `db.sample.insertMany([{name: "karroid1", password: "1111"}, {name: "karroid2", password: "1111"}])`

#### 자바스크립트의 반복문 사용 가능

```js
var num = 1
for (var i = 0; i < 3; i++) {db.sample.insertOne({name: "user" + i, score: num})}
```

### 2) 데이터 조회

#### 개요

- 조회를 할 때는 단일 Collection에서 동작
- cursor: 데이터를 조회할 때 결과 set을 cursor라고 부르고 반복적인 객체를 리턴 (iterator)


#### find()

- 형식 
    - `find(<query>, <projection>)`

    - `db.sample.find()`: 도큐먼트 전체를 조회

- 조건 검색

    - find({컬럼이름: 값})
    - name이 user1인 데이터 조회
    - `db.sample.find({name: "user1"})`
    - 여러 개의 컬럼을 나열하면 AND

    - 샘플 데이터 삽입
        ```js
        db.containerBox.insertMany([
            {name: "bear", weight: 60, category: "animal"},
            {name: "bear", weight: 10, category: "animal"},
            {name: "cat", weight: 2, category: "animal"},
            {name: "phone", weight: 1, category: "electronic"}
        ])
        ```

        ```js
        db.containerBox.find({name: "bear", category: "animal"})
        ```

#### 특정 컬럼 추출

- 두 번째 매개변수로 컬럼 이름과 true or false(1, 0)으로 설정하면 되는데 true면 조회가 되고 false이면 조회가 되지 않음
- _id는 false를 설정하지 않으면 기본적으로 조회가 됨
    
    - `db.containerBox.find({}, {name: true})`

- _id와 name을 제외하고 조회
    
    - `db.containerBox.find({}, {_id: false, name: false})`

#### 비교 오퍼레이터

```
- $eq: equal
- $ne: not equal
- $gt: greater than
- $gte: greater than or equals
- $lt: less than
- $lte: less than or equals
- $in: 배열 안에 속한 값
- $nin: 배열 안에 속하지 않은 값
```

- height 값이 175에서 180 사이
    - {height: {$gte: 175, $lte: 180}}

- inventory 컬렉션에서 item 컬럼의 값이 hello인 데이터 조회
    - `db.inventory.find({item: {$eq: "hello"}})`
- inventory 컬렉션에서 tags의 값이 blank나 blue인 데이터 조회
    - `db.inventory.find({tags: {$in: ["blank", "blue"]}})`
- inventory 컬렉션에서 tags의 값이 blank나 blue가 아닌 데이터 조회
    - `db.inventory.find({tags: {$nin: ["blank", "blue"]}})`

#### 논리 결합 오퍼레이터

```
$not
$or
$and
$nor: 주어진 조건 중 하나라도 만족하지 않는 데이터 조회
```

- inventory 컬렉션에서 qty가 2보다 크지 않은 데이터 조회
    - `db.inventory.find({qty: {$lte: 2}})`
    - `db.inventory.find({qty: {$not: {$gt: 2}}})`
- inventory 컬렉션에서 qty가 100보다 크거나 qty가 10보다 작은 데이터 조회
    - `db.inventory.find({$or: [{qty: {$gt: 100}, {qty: {$lt: 10}}}]})`
- inventory 컬렉션에서 price의 값이 0.99 또는 1.99이고 sale 필드의 값이 true 또는 qty의 값이 20 미만인 데이터 조회
    ```
    db.inventory.find({
        $and: [
            { $or: [{ price: 0.99 }, { price: 1.00 }] },
            { $or: [{ sale: true }, { qty: { $lt: 20 } }] }
        ]
    })
    ```

#### null 조회

- 실제 값이 null인 데이터도 조회가 되고 key가 없어도 null로 간주
    - `db.temp.insert({y: null})`
    - `db.temp.insert({y: 1})`

    - `db.temp.find({y: null})`
    - `db.temp.find({x: null})`

#### 정규식

- $regex를 이용

    - `{컬럼이름: {$regex:/pattern/, $options: '옵션'}}`

- option
    - i: 대소문자 무시
    - m: 정규식에 ^를 사용할 때 \n이 있으면 무시
    - x: 정규식 안에 있는 모든 공백을 무시
    - s: .을 사용할 때 \n을 포함해서 매치

    - `db.users.insert({name: "paulo"})`
    - `db.users.insert({name: "patric"})`
    - `db.users.insert({name: "pedro"})`

- a가 포함된 데이터 조회
    - `db.users.find({name: /a/})`

- pa로 시작하는 데이터를 조회
    - `db.users.find({name: /^pa/})`

- lo로 끝나는 데이터를 조회
    - `db.users.find({name: /lo$/})`

#### 배열 연산자

- $all: 순서와 상관없이 배열 안의 요소가 모두 포함되면 선택
- $elemMatch: 조건과 맞는 배열 요소를 가진 Document를 선택
- $size: 배열의 크기가 같은 Document를 선택

    ```
    db.inventory.insertMany([
        {item: "journal", qty: 25, tags: ["blank", "red"]},
        {item: "notebook", qty: 50, tags: ["red", "blank"]},
        {item: "paper", qty: 100, tags: []},
        {item: "planner", qty: 75, tags: ["blank", "red"]},
        {item: "postcard", qty: 45, tags: ["blue"]}
    ]);
    ```

- tags에 red가 포함된 데이터 조회: 
    - `db.inventory.find({tags: "red"}, {_id: false})`

- tags에 red와 blank가 포함된 데이터를 조회
    - `db.inventory.find({tags: ["red", "blank"]}, {_id: false})`
    - 배열의 순서가 다르면 다른 데이터

- tags에 red가 포함된 데이터 조회하는데 배열에서 일치하는 데이터만 조회
    - `db.inventory.find({tags: "red"}, {_id: false}, {"tags.$": true})`

- users 컬렉션에서 scores가 80보다 크고 90보다 작은 데이터가 1개라도 포함되어 있으면 조회
    ```
    db.users.find({
        scores: {
            $elemMatch: {$gt: 80, $lt: 90}
        }
    });
    ```

- tags에 red와 blank가 순서와 상관없이 포함하는 데이터를 조회
    ```
    db.inventory.find({
        tags: {
            $all: ["red", "blank"]
        }
    }, {_id: false});
    ```

- tags에 데이터가 없는 데이터를 조회
    ```
    db.inventory.find(
        {tags: {$size: 0}}, 
        {_id: false}
    );
    ```

- 필드 소유 여부: $exists에 true를 설정하면 필드가 있는 경우에만 조건을 확인
    ```
    db.inventory.find({
        qty: {
            $exists: true,
            $nin: [5, 15]
        }
    });
    ```

#### WHERE

- 일반적인 쿼리 연산자로 표현하기 어려운 복잡한 논리를 자바스크립트 표현식이나 함수를 통해서 실행할 때 사용

- 데이터베이스 내에서 자바스크립트 코드를 돌려 문서를 필터링

#### 서브 도큐먼트 필드 검색 쿼리

- RDBMS는 컬럼에 배열이나 다른 데이터를 삽입하는 것이 안되므로 이러한 쿼리가 존재하지 않음
    - `db.users.insert({name: "matt", contact: {type: "office", phone: "031-000-0000"}});`

- 조회를 할 때는 전체 조회도 가능하고 특정 필드를 기준으로 조회를 할 때는 .을 이용
    - `db.users.find({contact: {type:"office", phone: "031-000-0000"}});`

    - `db.users.find({"contact.type": "office"});`

#### 개수 제한은 limit 함수

- `db.inventory.find().limit(3)`


#### 한개만 검색: findOne 함수

- `db.inventory.findOne()`

#### 데이터를 건너뛰는 것은 skip()

- 첫번째를 건너뜀
- `db.inventory.find().skip(1)`

#### 정렬: sort()

- 컬럼 이름을 기재하고 1을 설정하면 오름차순이고 -1을 설정하면 내림차순

    - `db.inventory.find().sort({qty: 1})`
    - `db.inventory.find().sort({qty: -1})`

#### cursor

- 쿼리 결과에 대한 포인터로 find 함수를 호출하면 결과로 도큐먼트를 직접 반환하지 않고 cursor를 반환

- cursor에는 `hasNext()` 라는 함수가 존재하는데 다음 데이터 존재 여부를 리턴하고 next는 다음 데이터를 리턴한다.

#### 쿼리의 성능 측정

- find()를 호출하고 연달아서 `explain("executionStats")`을 호출하면 쿼리의 계획을 확인할 수 있음

- `db.inventory.find({qty: {$gte: 30, $lte: 90}}).explain("executionStats");`

#### 인덱스 설정

- createIndex({"필드이름": 1})
