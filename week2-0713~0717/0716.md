# MongoDB

## CRUD

### 1) 데이터 삽입 (도큐먼트 생성)

#### insert 의 두번쨰 매개변수로 ordered 옵션을 설정할 수 있는데 이 옵션에 true가 기본값인데 true가 설정되면 싱글 스레드로 동작하고 false로 설정하면 멀티 스레드를 이용해서 삽입한다.

- sample 컬렉션을 생성하는데 name의 값을 중복 없이 저장하도록 생성
    - `db.sample.createIndex({name: 1}, {unique: true})`

- 데이터 삽입
    - `db.sample.insert({name: "park"})`

- 여러 개의 데이터를 동시에 삽입
    - `db.sample.insert([{name: "kim"}, {name: "park"}, {name: "lee"}, {name: "choi"}])`

    - park이 중복돼서 에러가 발생
    - 지금은 싱글 스레드 형태로 삽입하므로 park이 에러가 나면 뒤의 2개의 데이터는 삽입되지 않는다.
    - `db.sample.find()`

- sample 컬렉션 삭제
    `db.sample.drop()`

- 멀티 스레드로 삽입
    - `db.sample.insert([{name: "kim"}, {name: "park"}, {name: "lee"}, {name: "choi"}], {ordered: false})`
    - park이 중복돼서 에러는 발생해도 나머지 데이터는 삽입됨

#### ObjectId

- MongoDB에서는 데이터를 삽입할 때 _id 항목의 값을 설정하지 않으면 Mongo DB가 ObjectId라는 자료형을 일련번호를 만들어서 자동 삽입
- 12byte로 구성되어 있고 new ObjectId()를 이용해서 직접 생성 가능

    ```js
    var newId = new ObjectId()
    db.sample.insert({_id: newId, name: "user01"})
    ```

#### insertOne과 insertMany
- insert 함수는 deprecated
- 하나의 데이터를 삽입할 때는 insertOne 그리고 여러 개의 데이터를 삽입할 때는 insertMany를 사용하도록 권장
    - `db.sample.insertOne({name: "karroid"}, {password: "1111"})`

    - `db.sample.insertMany([{name: "karroid1", password: "1111"}, {name: "karroid2", password: "1111"}])`

#### 자바스크립트의 반복문 사용 가능

```js
var num = 1
for (var i = 0; i < 3; i++) {db.sample.insertOne({name: "user" + i, score: num})}
```

### 2) 데이터 조회

#### 개요

- 조회를 할 때는 단일 Collection에서 동작
- cursor: 데이터를 조회할 때 결과 set을 cursor라고 부르고 반복적인 객체를 리턴 (iterator)


#### find()

- 형식 
    - `find(<query>, <projection>)`

    - `db.sample.find()`: 도큐먼트 전체를 조회

- 조건 검색

    - find({컬럼이름: 값})
    - name이 user1인 데이터 조회
    - `db.sample.find({name: "user1"})`
    - 여러 개의 컬럼을 나열하면 AND

    - 샘플 데이터 삽입
        ```js
        db.containerBox.insertMany([
            {name: "bear", weight: 60, category: "animal"},
            {name: "bear", weight: 10, category: "animal"},
            {name: "cat", weight: 2, category: "animal"},
            {name: "phone", weight: 1, category: "electronic"}
        ])
        ```

        ```js
        db.containerBox.find({name: "bear", category: "animal"})
        ```

#### 특정 컬럼 추출

- 두 번째 매개변수로 컬럼 이름과 true or false(1, 0)으로 설정하면 되는데 true면 조회가 되고 false이면 조회가 되지 않음
- _id는 false를 설정하지 않으면 기본적으로 조회가 됨
    
    - `db.containerBox.find({}, {name: true})`

- _id와 name을 제외하고 조회
    
    - `db.containerBox.find({}, {_id: false, name: false})`

#### 비교 오퍼레이터

```
- $eq: equal
- $ne: not equal
- $gt: greater than
- $gte: greater than or equals
- $lt: less than
- $lte: less than or equals
- $in: 배열 안에 속한 값
- $nin: 배열 안에 속하지 않은 값
```

- height 값이 175에서 180 사이
    - {height: {$gte: 175, $lte: 180}}

- inventory 컬렉션에서 item 컬럼의 값이 hello인 데이터 조회
    - `db.inventory.find({item: {$eq: "hello"}})`
- inventory 컬렉션에서 tags의 값이 blank나 blue인 데이터 조회
    - `db.inventory.find({tags: {$in: ["blank", "blue"]}})`
- inventory 컬렉션에서 tags의 값이 blank나 blue가 아닌 데이터 조회
    - `db.inventory.find({tags: {$nin: ["blank", "blue"]}})`

#### 논리 결합 오퍼레이터

```
$not
$or
$and
$nor: 주어진 조건 중 하나라도 만족하지 않는 데이터 조회
```

- inventory 컬렉션에서 qty가 2보다 크지 않은 데이터 조회
    - `db.inventory.find({qty: {$lte: 2}})`
    - `db.inventory.find({qty: {$not: {$gt: 2}}})`
- inventory 컬렉션에서 qty가 100보다 크거나 qty가 10보다 작은 데이터 조회
    - `db.inventory.find({$or: [{qty: {$gt: 100}, {qty: {$lt: 10}}}]})`
- inventory 컬렉션에서 price의 값이 0.99 또는 1.99이고 sale 필드의 값이 true 또는 qty의 값이 20 미만인 데이터 조회
    ```
    db.inventory.find({
        $and: [
            { $or: [{ price: 0.99 }, { price: 1.00 }] },
            { $or: [{ sale: true }, { qty: { $lt: 20 } }] }
        ]
    })
    ```

#### null 조회

- 실제 값이 null인 데이터도 조회가 되고 key가 없어도 null로 간주
    - `db.temp.insert({y: null})`
    - `db.temp.insert({y: 1})`

    - `db.temp.find({y: null})`
    - `db.temp.find({x: null})`

#### 정규식

- $regex를 이용

    - `{컬럼이름: {$regex:/pattern/, $options: '옵션'}}`

- option
    - i: 대소문자 무시
    - m: 정규식에 ^를 사용할 때 \n이 있으면 무시
    - x: 정규식 안에 있는 모든 공백을 무시
    - s: .을 사용할 때 \n을 포함해서 매치

    - `db.users.insert({name: "paulo"})`
    - `db.users.insert({name: "patric"})`
    - `db.users.insert({name: "pedro"})`

- a가 포함된 데이터 조회
    - `db.users.find({name: /a/})`

- pa로 시작하는 데이터를 조회
    - `db.users.find({name: /^pa/})`

- lo로 끝나는 데이터를 조회
    - `db.users.find({name: /lo$/})`

#### 배열 연산자

- $all: 순서와 상관없이 배열 안의 요소가 모두 포함되면 선택
- $elemMatch: 조건과 맞는 배열 요소를 가진 Document를 선택
- $size: 배열의 크기가 같은 Document를 선택

    ```
    db.inventory.insertMany([
        {item: "journal", qty: 25, tags: ["blank", "red"]},
        {item: "notebook", qty: 50, tags: ["red", "blank"]},
        {item: "paper", qty: 100, tags: []},
        {item: "planner", qty: 75, tags: ["blank", "red"]},
        {item: "postcard", qty: 45, tags: ["blue"]}
    ]);
    ```

- tags에 red가 포함된 데이터 조회: 
    - `db.inventory.find({tags: "red"}, {_id: false})`

- tags에 red와 blank가 포함된 데이터를 조회
    - `db.inventory.find({tags: ["red", "blank"]}, {_id: false})`
    - 배열의 순서가 다르면 다른 데이터

- tags에 red가 포함된 데이터 조회하는데 배열에서 일치하는 데이터만 조회
    - `db.inventory.find({tags: "red"}, {_id: false}, {"tags.$": true})`

- users 컬렉션에서 scores가 80보다 크고 90보다 작은 데이터가 1개라도 포함되어 있으면 조회
    ```js
    db.users.find({
        scores: {
            $elemMatch: {$gt: 80, $lt: 90}
        }
    });
    ```

- tags에 red와 blank가 순서와 상관없이 포함하는 데이터를 조회
    ```js
    db.inventory.find({
        tags: {
            $all: ["red", "blank"]
        }
    }, {_id: false});
    ```

- tags에 데이터가 없는 데이터를 조회
    ```js
    db.inventory.find(
        {tags: {$size: 0}}, 
        {_id: false}
    );
    ```

- 필드 소유 여부: $exists에 true를 설정하면 필드가 있는 경우에만 조건을 확인
    ```js
    db.inventory.find({
        qty: {
            $exists: true,
            $nin: [5, 15]
        }
    });
    ```

#### WHERE

- 일반적인 쿼리 연산자로 표현하기 어려운 복잡한 논리를 자바스크립트 표현식이나 함수를 통해서 실행할 때 사용

- 데이터베이스 내에서 자바스크립트 코드를 돌려 문서를 필터링

#### 서브 도큐먼트 필드 검색 쿼리

- RDBMS는 컬럼에 배열이나 다른 데이터를 삽입하는 것이 안되므로 이러한 쿼리가 존재하지 않음
    - `db.users.insert({name: "matt", contact: {type: "office", phone: "031-000-0000"}});`

- 조회를 할 때는 전체 조회도 가능하고 특정 필드를 기준으로 조회를 할 때는 .을 이용
    - `db.users.find({contact: {type:"office", phone: "031-000-0000"}});`

    - `db.users.find({"contact.type": "office"});`

#### 개수 제한은 limit 함수

- `db.inventory.find().limit(3)`


#### 한개만 검색: findOne 함수

- `db.inventory.findOne()`

#### 데이터를 건너뛰는 것은 skip()

- 첫번째를 건너뜀
- `db.inventory.find().skip(1)`

#### 정렬: sort()

- 컬럼 이름을 기재하고 1을 설정하면 오름차순이고 -1을 설정하면 내림차순

    - `db.inventory.find().sort({qty: 1})`
    - `db.inventory.find().sort({qty: -1})`

#### cursor

- 쿼리 결과에 대한 포인터로 find 함수를 호출하면 결과로 도큐먼트를 직접 반환하지 않고 cursor를 반환

- cursor에는 `hasNext()` 라는 함수가 존재하는데 다음 데이터 존재 여부를 리턴하고 next는 다음 데이터를 리턴한다.

#### 쿼리의 성능 측정

- find()를 호출하고 연달아서 `explain("executionStats")`을 호출하면 쿼리의 계획을 확인할 수 있음

- `db.inventory.find({qty: {$gte: 30, $lte: 90}}).explain("executionStats");`

#### 인덱스 설정

- createIndex({"필드이름": 1})

### 3) 데이터 집계

#### 개요

- 저장되어 있는 정보에 다른 정보를 합해서 출력하거나 그룹화를 통해서 다른 형태로 정보를 생성해서 리턴하는 작업

- 집계 방법


|          | 데이터를 가져와서 애플리케이션에서 수행 | Map-Reduce   | Pipeline 활용 |
| -------- | ------------------------------| -----------  | -------------- |
| 자유도     | 좋음                           | 좋음          | 나쁨           |
| 처리속도   | 가장느림                         | 보통          | 가장 좋음      |
| 메모리     | 사용량이 많음                    | 사용량이 많음    | 사용량이 적음    |
| 처리방식   | 애플리케이션 내부에서               | 자바스크립트 엔진 | MongoDB 내부   |

'manufacture':"Sony", 'price':499.00})
db.products.insert({'name':'Macbook Air 13inch', 'category':'Laptops',

#### Map Reduce

- 기본 형식

```js
db.컬렉션.mapReduce(map, reduce, {
    out: <collection>,
    query: <document>,
    sort: <document>,
    limit: <number>,
    finalize: <function>,
    scope: <document>,
    isMode: <boolean>,
    verbose: <boolean>
})
```

- 동작 순서

query(필터링) -> sort(정렬) -> limit(개수 제한) -> Map 함수 수행 -> 그룹화 -> Reduce 함수 적용 -> finalizer 함수 -> out

```js
db.rating.insertMany([
{_id: 1, rating: 1, user_id: 2},
{_id: 2, rating: 2, user_id: 3},
{_id: 3, rating: 3, user_id: 4},
{_id: 4, rating: 3, user_id: 1},
{_id: 5, rating: 4, user_id: 5},
{_id: 6, rating: 4, user_id: 8},
{_id: 7, rating: 5, user_id: 9},
{_id: 8, rating: 5, user_id: 10},
{_id: 9, rating: 5, user_id: 11},
{_id: 10, rating: 5, user_id: 12}
])
```

- Map 함수

```js
var mapper = function() {
    emit(this.rating, this.user_id)
}
```

- reduce 함수

```js
var reducer = function(key, values) {return values.length}
```

- 수행

```js
db.rating.mapReduce(mapper, reducer, {out:{inline:1}})
```

#### 집계 파이프라인

- 데이터 개수는 countDocuments()
    - `db.products.countDocuments();`

- 중복 제거는 `distinct("필드이름")`
    - `db.products.distinct('manufacture')`

- `aggregate()`

    - $group 으로 그룹화 할 항목을 설정
    - $sum, min, max, avg, first, last, push, addToSet 등의 집계함수 설정

- products 에서 manufacture 별로 price 의 합계

    ```js
    db.products.aggregate([{
        $group:{
            _id:{
                "maker": "$manufacture"
            },
            sum_prices:{
                $sum: "$price"
            }
        }
    }])
    ```

- 세부 그룹화 - manufacture 별로 그룹화하고 category 별로 세부 그룹화

    ```js
    db.products.aggregate([{
        $group:{
            _id:{
                "maker": "$manufacture"
            },
            categories:{
                $push: "$category"
            }
        }
    }])
    ```

- 정렬
    ```js
    db.products.aggregate([{
        $sort:{
            price: -1
        }
    }])
    ```

### 4) 데이터 수정 및 삭제

#### 갱신을 위한 함수

- `update()`
    - 매개변수는 조건, 갱신 내용, 옵션 3가지
    ```js
    // 초기화
    db.sample.drop();

    db.sample.insert({name: "adam", score: 90})
    ```
    ```js
    db.sample.update(
        {name: "adam"},
        {$set: {score: 100}}
    )
    ```
    - 수정 연산자
        - `$set`
        - `$currentDate: 필드`: 현재 날짜로 설정
        - `$inc`
        - `$min`
        - `$max`
        - `$mul`
        - `$rename`
        - `$setOnInsert`
        - `$unset`: 컬럼 제거

    ```js
    // 점수가 10점 줄어든다.
    db.sample.update(
        {name: "adam"},
        {$inc: {score: -10}}
    )
    ```

    ```js
    // 컬렴 이름 변경
    db.sample.update(
        {name: "adam"},
        {$rename: {score: "점수"}}
    )
    ```
    
    - 배열 수정
        - `$addToSet`: 배열 안에 해당하는 값이 없으면 추가
        - `$pop`: 배열의 첫번째 또는 마지막 요소를 삭제
        - `$pull`: 쿼리에 해당하는 요소 하나를 제거
        - `$push`: 해당 요소를 배열에 추가
        - `$pullAll`: 해당하는 값을 가진 요소 전부를 제거

    ```js
    // 초기 세팅
    db.board.insert(
        {
            "board_num": 1,
            "board_id": "ggangpae1",
            "board_title": "가입인사",
            "board_content": "반갑습니다",
            "reply": []
        }
    )
    ```
    ```js
    // reply 수정
    db.board.update(
        {"board_num": 1},
        {"$push":{
            "reply": {
                "reply_num": 1,
                "reply_content": "안녕",
                "reply_id": "root"
            }
        }}
    )
    ```
    ```js
    db.board.update(
        {
            "board_num": 1,
            "reply.reply_num": 1
        },
        {"$set":{
            "reply.$": {
                "reply_content": "반가워"
            }
        }}
    )
    ```
    ```js
    // 지우기
    db.board.update(
        {
            "board_num": 1
        },
        {"$pull":{
            "reply": {
                "reply_num": 1
            }
        }}
    )
    ```

- `updateOne()`
    - 조건에 맞는 데이터 1개만 수정
- `updateMany()`
    - 조건에 맞는 데이터 모두를 수정

- `replaceOne()`

    ```js
    db.collection이름.replaceOne(
        <query>,
        <replacement - 수정할 내용>
        {
            upset: <boolean>,
            writeConcern: <document>,
            collation: <document>
        }
    )
    ```

    ```js
    db.users.replaceOne(
        {name: "matt"},
        {
            username:"karoid",
            status: "sleep",
            points: 100,
            password:2222
        }
    )
    ```

    ```js
    db.users.replaceOne(
        {name: "matt"},
        {
            username:"karoid",
            status: "sleep",
            points: 100,
            password:2222
        },
        {upset: true}
    )
    ```

#### 데이터 삭제

- 삭제를 위한 함수
    - `deleteOne`
    - `deleteMany`
        - `db.users.deleteMany({})`
    - `remove`

- 조건이 없으면 `deleteMany`는 모든 데이터를 삭제

- 데이터를 삭제하거나 수정할 때 외부에서 접근할 수 없도록 할 수 있는데 이 때는 `$isolated:1` 옵션을 추가해주면 된다.

# Redis

## Redis

### 1) Redis(Remote Dictionary Server)

#### 특징

- key-value Data Store
- 싱글 스레드로 동작
- 속도가 매우 빠름: In Memory Database

    - MariaDB나 MongoDB는 On Disk Database

#### 고가용성

- 언제든지 사용할 수 있어야한다 라는 성질
- 자체적으로 HA(high availability) 기능 제공
- 복제를 통해서 여러 서버에 분산시킬 수 있으며 sentinel은 장애 상황을 자동으로 탐지해서 자동으로 fail-over(시스템에 장애가 발생했을 떄 예비 시스템으로 업무를 즉시 전환하여 서비스 중단없이 운영을 유지하는 기술)를 수행

- 애플리케이션은 노드에 직접 접속해서 데이터를 받아오는 것이 아니고 Sentinel을 이용해서 데이터를 받아오기 때문에 마스터 노드에 장애가 발생하더라도 엔드포인트를 변경할 필요가 없다. 

#### 확장성

- 레디스를 클러스터 모드로 구성하면 손쉬운 수평 확장이 가능
- 데이터는 레디스 클러스터 내에서 자동으로 샤딩된 후 저장되며 여러 개의 복제본을 만들어서 저장
- 애플리케이션에서는 실제 데이터가 어떤 노드에 있는지 몰라도 사용 가능: 위치 투명성
- 클러스터 구조에서는 모든 레디스 인스턴스는 클러스터 버스라는 프로토콜을 이용해서 서로 감시하고 있으며 이를 이용해서 클러스터의 마스터 노드에 문제가 발생하면 자동으로 fail-over를 이용해서 고가용성을 유지

#### 대다수의 클라우드 벤더들이 서비스를 제공

- AWS: Amazon ElastiCache for Redis
- GCP: Cloud Memory store for Rdeis
- Azure, NHN, Naver Cloud에서도 제공

### 2) MicroService와 Redis

#### 데이터 저장소로서의 Redis

- 서비스 별 개별 저장소로 사용하기에 적합
- 설치가 간편
- 다양한 자료구조를 제공하고 사용이 간단
- 고가용성을 위해서 로드밸런서나 프록시 등 추가적인 서비스를 설치할 필요가 없음
- AOF(Append Only File) 과 RDB(Redis DataBase) 형식으로 디스크에 저장 가능

#### Message Broker로서의 레디스

- 마이크로 서비스에서는 특별한 경우가 아니면 Message Broker가 필요
- Message Broker는 서비스들 간의 비동기 데이터 전달을 수행
- pub/sub 기능을 이용
- 모든 데이터는 전달된 뒤 삭제가 되므로 모든 메시지 상황에 사용할 수는 없지만 fire-and-forget 패턴이 필요한 간단한 알림 서비스에서 유용하게 사용됨.

- stream과 list 자료구조를 이용해서 구현이 가능한데 stream은 Apache Kafka에서 영감을 얻어서 추가

### 3) 반영구적 저장

#### RDB(Redis DataBase)
- 현재 메모리에 있는 데이터 전체에 해단 스냅샷을 작성하고 이를 디스크에 저장하는 방식
- 간단하게 사용이 가능
- 스냅샷 이후에 변경된 데이터는 복구가 안됨

#### AOF(Append Only File)
- Redis에 데이터가 변경되는 이벤트가 발생하면 이를 모두 저장하는 방식
- 데이터를 복구할 떄 RDB 방식보다 유실량이 적음
- 속도가 느리고 로그의 크기가 커짐

### 4) 이벤트 루프

#### 클라이언트가 실행한 명령어들을 이벤트 큐에 저장한 후 싱글 스레드로 하나씩 처리하는 방식

#### 멀티 스레드 방식의 문제점

- 여러 개의 스레드를 동시에 수행하다 보면 Context Switching (문맥 교환)이 발생
    - 스레드가 자신의 작업을 수행하다가 다른 스레드에게 제어권을 넘길 때 현재 작업 상태를 저장하는 것을 문맥 교환

- 여러 개의 스레드가 동시에 공유 자원을 사용하려고 하다보면 Dead Lock이 발생할 수 있음.

#### 싱글 스레드는 멀티 스레드보다 작업 처리 속도가 느림

### 5) 장점

- 빠른 성능: 인 메모리 데이터 구조
- 다양성과 사용 편의성
- 대다수의 개발 언어를 지원
- 복제 및 지속성

### 6) 사용 사레

-  주 데이터 저장소

-  Caching
    - 다른 데이터베이스 앞에 두고 자주 사용하는 데이터를 Redis에 저장해서 애플리케이션이 데이터를 읽어오는 시간을 줄임

-  세션 관리

    - 세션: 서버의 메모리에 저장하는 정보
    - 게임, 전자상거래, 소셜 미디어 플랫폼에서 주로 이용

-  실시간 순위표

-  분산락: 여러 대의 컴퓨터가 공유 자원을 사용할 때 실제 사용 중인지 여부를 판단하기 위한 Lock
-  메시지 브로커
-  채팅 및 메시징

### 7) Architecture

#### Redis는 Sentinel, Master, Replica로 구성
#### Sentiel이 Master와 Replica를 관리
#### Sentinel이 주기적으로 서버들을 모니터링하고 Master가 서비스할 수 없는 상태가 되면 Replica 중 하나를 Master 서버로 변경

#### Redis Sentinel도 장애가 발생할 수 있으므로 여러 대의 서버로 구성

#### 클러스터 아키텍쳐

- 클러스터를 구성하면 클러스터에 포함된 모든 노드들이 서로 통신하면서 고가용성을 유지
- 샤딩 기능이 기본 기능
- 클러스터 구조를 이용하면 Sentinel을 설정할 필요가 없음
- 클러스 내부의 모든 노드들이 Mesh 형으로 연결
- 샤딩을 할 때 해시 함수를 이용해서 데이터를 분배하기 때문에 데이터가 균등하게 저장
- 노드를 추가하거나 삭제하면 자동으로 재분배(Re-Balancing 이나 Re-shard)
- 애플리케이션 클라이언트는 Redis 클러스터의 노드 중 하나라도 연결되면 클러스터의 전체 상태 정보를 확인할 수 있음(클러스터 내부의 노드는 전체 노드 정보를 알 수 있음)
- 클러스터에 한 번 접속한 클라이언트는 장애가 발생한 노드나 확장을 위해 추가한 노드 정보들도 모두 업데이트 받기 때문에 실제 Production 환경에서 운영 중 증설을 하더라도 애플리케이션의 재시작 없이 대응 가능
