## 3. Shell

### 7) 쉘 내장 명령

- 쉘은 자체적으로 내장 명령을 가지고 있는데 일반적인 리눅스 명령어들은 `/bin` 이나 `/usr/bin` 또는 `/usr/sbin` 디렉토리에서 별도의 실행 파일로 제공하지만 쉘 명령은 별도의 실행 파일 없이 쉘 안에 포함되어 있음


- **명령어 확인**
    - pwd 명령어의 파일을 확인
        - `file /usr/bin/pwd`
        - `whereis pwd`

    - cd 명령의 파일을 확인
        - `whereis cd`
        - 경로가 출력되지 않는데 이는 쉘 내장명령이기 때문

- **echo**
    - 화면에 한 줄의 문자열을 출력
    - 형식
        - `echo [-n] [문자열 또는 변수]`
        - n은 마지막에 줄 바꿈을 하지 않음
    - 문자열 출력
        - `echo linux`
        - `echo "ubuntu linux"`
        - 중간에 공백이 있을 때는 묶어주는 것이 좋은데 묶어주지 않으면 앞의 내용이 옵션이 될 수도 있기 때문

- **printf**
    - 데이터를 형식화해서 화면에 출력
    - 형식
        - `printf [형식] [인수]`
    - C언어의 printf 함수와 사용법이 동일
        - `printf "%d + %d = %d\n" 2 3 5`

### 8) 특수문자

- **개요**
    - 쉘은 사용자가 더욱 편리하게 명령을 입력하고 실행할 수 있도록 다양한 특수 문자를 제공
    - 특수 문자의 종류와 사용법은 거의 모든 쉘에서 비슷
    - `*, ?, |, [], ~`
    - 사용자가 명령을 입력하면 입력한 내용 중에 특수 문자가 있는지 확인해서 이를 해독해서 변경한 후 명령을 실행

- `*`
    - 임의의 문자열을 의미
        - `ls *`: 현재 디렉토리의 모든 파일과 서브 디렉토리를 나열
        - `cp * /tmp`: 현재 디렉토리의 모든 내용을 `/tmp` 로 복사

- `?` 
    - 하나의 문자를 의미
        - `ls t?.txt`: t로 시작하는 2글자 텍스트 파일을 조회
        - `-`를 이용하면 범위를 설정

        - `ls -l tmp[1 3 5].txt`: 1, 3, 5 중 하나
        - `[1 3 5]`: 1부터 3까지
        - `[A-Za-z]`: 영문 한 글자
        - `[가-힣]`: 한글 한 글자

- `~`: 홈 디렉토리
- `-`: 현재 디렉토리로 이동하기 전의 디렉토리

- ``` `(백틱) ```
    - 명령을 수행해서 명령의 결과를 대체

    - ``` echo "Today is `date`" ```

- `;, |, &&`
    - `;`: 여러 개의 명령어를 순차적으로 실행할 때 사용하는데 앞의 명령어가 실패해도 다음 명령어를 수행
    - `&&`: 앞의 명령어가 성공했을 때 뒤의 명령어를 수행
    - `|`: 앞 명령어의 실행 결과를 뒤 명령어에 전달, Pipe

        - ```date; ls```: 2개의 명령을 순차적으로 수행
        - `date&&ls`: date 명령이 성공적으로 수행되면 ls 수행
        - `ls -al / | more`: `ls -al /` 명령의 결과를 `more`에 전달

- `\`
    - 특수 문자의 효과를 무력화
    - `cat t*`: t로 시작하는 2글자 파일의 내용을 출력
    - `cat t\*`: t* 이라는 파일의 내용을 출력

    - `''`로 감싸면 모든 특수문자의 기능이 무력화
    - `""`로 감싸면 `$, \, ${}, {}` 은 원래대로 처리


### 9) 표준 입출력

- **종류**

|파일 디스크립터 | 파일 디스크립터 대신 사용하는 이름      | 정의     |
|------------|---------------------------------|---------|
| 0          |       stdin                     | 표준 입력 |
| 1          |       stdout                    | 표준 출력 |
| 2          |       stderr                    | 표준 오류 |

- **출력 redirection**
    - 출력의 방향을 변경하는 것
    - 기본적으로 출력은 모니터
    - 형식
        - `명령 1 > 경로`
        - `명령 > 경로`

- **date 명령의 결과를 파일에 저장**

    - date: 표준 출력 장치인 모니터로 출력
    - `date 1>print.txt`
    - `cat print.txt` 로 확인

    - 파일에 내용을 추가 > 대신에 >>

- **오류 리다이렉션**
    - 오류 메시지를 모니터가 아닌 다른 곳에 출력

    - 형식
        `명령 2>경로`

    - 에러가 발생하는 명령을 1로 리다이렉션하면 성공했을 때는 기록을 하지만 실패했을 때는 기록하지 않음
        - `ls a 2>err.txt`
    - 오류 메시지 버리기
        - `2>/dev/null`
    - 성공과 실패 모두 기록하기
        - 명령어 > 파일경로 2>&1
        - `ls >ls.out 2>&1`
        - ls >ls.out 2>&1 실행 순서:
            - ls.out 먼저 처리 → fd 1(stdout)이 ls.out 파일을 가리키게 됨
            2>&1 처리 → fd 2(stderr)를 "현재 fd 1이 가리키는 곳"으로 연결 → 그 시점에 fd 1은 ls.out을 가리키고 있으므로, fd 2도 결국 ls.out을 가리키게 됨

- **입력 리다이렉션**

- 표준 입력을 변경하는 것
- 표준 입력은 키보드
- 표준 입력을 파일로 변경하는 것

### 10) 변수

- __종류__
    - 쉘 변수: 현재 쉘에서만 사용 가능한 변수
    - 환경 변수: 시스템 전체에 적용되는 변수

- __변수 확인__
    - `echo $변수명`
    - `env`: 환경 변수 전부 출력
    - `set`: 모든 변수와 함수를 출력

- __변수 생성 및 수정__
    - 쉘 변수 생성 및 수정: 없으면 생성되고 있으면 수정
        - `쉘변수이름=값`
    - 환경 변수 생성 및 수정
        - `export 변수이름=값`

- __변수 삭제__
    - `unset 변수명`

- __자주 사용하는 환경 변수__
    - `HOME`: 현재 사용자의 홈 디렉토리
    - `LANG`: 기본 지원되는 언어
    - `PATH`: 실행 파일을 찾는 디렉토리 경로
    - `PWD`: 현재 작업 중인 디렉토리

- __실습__
    - 기존 환경 변수 출력
    - `echo $PATH`

    - 변수 생성
        - `SOME=test`
        - `echo $SOME`

- __프롬프트 설정: PS1__
    - 프롬프트에서 사용할 수 있는 기호
        - `[` 와 `@`: 그대로 출력
        - `\u`: 현재 사용자
        - `\h`: 호스트 이름
        - `\W`: 현재 위치의 절대 경로 가운데 현재 디렉토리 이름만을 출력
        - `\\$`: root이면 #을 표시하고 일반 사용자라면 $를 표시

    - 프롬프트 모양을 변경할 때는 프롬프트 모양을 저장하고 있는 `PS1` 이라는 환경변수를 활용
        - `PROMPT=$PS1`
        - `PS1='LINUX]'`
        - `PS1=PROPMT` : 다시 원상복구

- __PATH__
    - 명령어의 위치를 찾는 디렉토리를 저장하고 있는 변수
    - 각각의 디렉토리를 :으로 구분
    - PATH 확인
        - `echo $PATH`
    - PAHT에 경로 추가
        - `PATH="$PATH:추가할경로"`
    - 현재 디렉토리의 bin 이라는 디렉토리를 PATH에 추가
        - `PATH="$PATH:~/bin"`

- __LANG__
    - 로케일 정보를 저장하고 있는 변수
    - 이 변수에 로케일 값을 설정하면 로케일이 수정
    - 한글을 사용하고자 할 때 이 변수를 사용
    - `echo $LANG`: 현재 사용 중인 언어

### 11) 환경 설정 파일

- 시스템을 사용하는 사용자의 환경을 설정하는 파일로 로그인할 때마다 무조건 실행되는 파일

- __시스템 환경 설정 파일__
    - `/etc/profile`
        - 모든 쉘에 공통 적용되는 환경 설정 파일
        - 아래 2개의 파일을 실행
    - `/etc/bash.bashrc`
        - 기본 프롬프트를 설정
        - sudo 명령과 관련된 힌트를 설정
    - `/etc/profile.d/*.sh`
        - 언어나 명령 별로 필요한 환경을 설정

- __사용자 환경 설정 파일__
    - `~/.profile`: 경로 추가 등 사용자가 정의하는 환경 설정 파일
    - `~/.bashrc`: 히스토리의 크기를 설정하거나 기본 별명이나 함수를 설정

    - `~/.bash_logout`: 로그아웃할 때 실행할 필요가 있는 함수를 설정

    - `~/.bash_aliases`: 사용자가 정의한 별명 설정 파일
    - 이 파일들을 수정했을 떄 적용은 로그인을 다시 하던가 `.` 이나 `source` 명령을 이용해서 적용

- __별명 설정 파일을 수정하고 적용__
    - 파일에 작성: `sudo vi ~/.bash_aliases`

    ```
    alias rm='rm -i'
    alias h=history
    alias c=clear
    ```

    - 적용
        - `source ~/.bash_aliases`

- __set 명령__
    - `-o` 나 `+o` 를 지정하면 옵션 기능을 활성화하거나 비활성화
    - `noclobber` 라는 옵션을 설정하면 파일에 덮어쓰기가 안된다.

```sh
touch file1 # 파일생성
echo 12345 > file1 # file1에 기록
set -o noclobber # 비활성화
echo 12345 > file1 # file1에 기록이 되지 않음
set -o noclobber # 다시 활성화
```

## 4. File System

### 1) 개요

- 유닉스 계열의 운영체제이므로 유닉스의 많은 부분을 그대로 이어받아서 유지
- 유닉스에서는 시스템과 관련된 정보와 하드웨어 같은 장치를 모두 파일로 관리
- 파일은 관련있는 정보들의 집합
- 파일을 효과적으로 관리하기 위해서 디렉토리를 사용
- 윈도우는 물리적인 디스크가 여러 개라면 디렉토리 트리도 여러 개지만 리눅스에서는 시스템 전체에 단 하나의 트리만 가짐
    - 윈도우에는 물리적인 디스크가 2개라면 C:\와 D:\ 2개의 디렉토리 트리가 존재
    - 유닉스는 /가 최상위 디렉토리

- 우분투에서 관리하는 모든 데이터는 파일로 인식 (현재 실행 중인 프로세스, 일반 파일, 하드웨어 등)
- 하나의 프로세스는 하나의 작업 디렉토리를 가짐
- 우분투는 일반 파일, 디렉토리 파일, 장치 파일, 심볼릭 링크를 가짐

### 2) 파일의 종류

- __일반 파일__
    - 데이터를 관리하는데 주로 사용되는 일상적인 파일을 의미
    - 실행 파일이나 텍스트 파일, 이미지 파일, 프로그램 소스 파일 등 우분투에서 사용하는 대부분의 파일을 의미하며 실행 파일이나 이미지 파일의 경우는 데이터가 바이너리 형태로 저장

    - `바이너리 파일`
        - 문자가 아닌 데이터 파일을 의미하고 기계어인 0과 1의 값으로 구성
        - 프로그램을 코딩해서 컴파일을 수행하는 과정에서 생성되는 파일을 의미하며 실행이라고도 부름

- __디렉토리 파일__
    - 디렉토리도 하나의 파일로 관리
    - 디렉토리 파일은 해당 디렉토리에 저장된 파일이나 하위 디렉토리에 대한 정보를 소유
    - 디렉토리는 일반 파일과 마찬가지로 디스크에 저장돼서 다른 파일을 조작하고 엑세스하는데 필요한 정보를 소유

- __링크 파일__
    - 사용자에게 편리성을 제공해주는 파일
    - 리눅스에서는 link 개념을 이용해서 여러 개의 파일 이름이 하나의 i-node를 가리키도록 해서 실제 파일이나 디렉토리 또는 다른 링크를 가리키도록 한다. 

    - `하드 링크`
        - 원본 파일을 복사해서 원본 파일과 동일한 내용의 다른 사본 파일을 만드는 것
        - 하드 링크에서는 원본 파일과 링크 파일 2개가 서로 다른 파일로 취급돼서 하나의 파일이 지워지더라도 다른 파일은 남아있다. 
        - 하드 링크에서는 원본 파일의 내용이 변경될 경우에는 복사된 파일의 내용도 자동으로 변경
    - `심볼릭 링크`
        - 단축아이콘처럼 원본 파일이 삭제되면 같이 삭제가 되는 형태의 링크 파일

- __장치 파일(특수 파일)__
    - 하드디스크나 키보드 같은 장치도 파일로 취급
    - 리눅스 시스템에 부착된 장치들을 관리하기 위한 특수 파일
    - 대부분의 장치 파일은 /dev 디렉토리에 존재

### 3) 파일의 종류 확인

- `file` 명령으로 확인 가능
    - .profile 이라는 파일의 종류 확인
        - `file .profile`
    - 현재 디렉토리 확인
        - `file .`
    - /bin/bash를 확인: 실행 파일
        - `file /bin/bash`

### 4) 디렉토리와 파일 관리

- __개요__
    - 우분투 파일 시스템은 파일과 디렉토리의 계층형 구조로 구성(Tree)
    - 우분투에서는 `최상위 디렉토리(/)`를 하나 두고 그 아래에 디렉토리나 파일을 배치
    - __현재 작업 디렉토리__
        - `.`으로 표시
        - 절대 경로를 알고자 할 때는 `pwd` 명령을 사용하면 된다.

    - __상위 디렉토리__
        - 현재 작업 중인 디렉토리를 포함하고 있는 디렉토리
        - `..`으로 표시
    - __홈 디렉토리__
        - 각 사용자에게 할당되는 디렉토리
        - 계정을 만들 때 지정이 가능
        - 일반적으로는 `/home/계정` 으로 생성
        - `~`로 표시

- __기본적으로 제공되는 디렉토리__
    - `ls -F /`
    - `/`로 끝나는 이름은 디렉토리
    - `@`로 끝나는 이름은 심볼릭 링크

    - bin과 lib는 유닉스 초기부터 있었던 디렉토리로 지금은 그 기능이 /usr/bin 과 /usr/lib로 통합되었으나 호환성 유지 문제 때문에 심볼릭 링크로 제공
    - 루트 계정에서 제공하는 디렉토리
        - `/dev`: 디바이스 파일이 배치되어 있는 디렉토리
        - `/home`: 사용자 별로 할당되는 홈 디렉토리가 배치되는 디렉토리
            - adam 이라는 사용자의 홈 디렉토리는 `/home/adam`
        - `/media`: CD나 USB와 같은 외부 장치를 연결하는 디렉토리
        - `/etc`: 리눅스 설정과 관련된 디렉토리
        - `/opt`: 추가 패키지가 설치되는 디렉토리

        - `/root`: root 계정의 홈 디렉토리
        - `/sys`: 리눅스 커널과 관련된 파일이 저장되는 디렉토리
        - `/usr`: 기본 실행 파일과 라이브러리 파일, 헤더 파일이 저장되는 디렉토리
        - `/boot`: 부팅 관련
        - `/lost+found`: 파일 시스템에 문제가 발생하여 복구할 경우 문제가 되는 파일이 저장되는 디렉토리
        - `/mnt`: 파일 시스템을 임시로 마운트하는 디렉토리
        - `/proc`: 프로세스 정보 및 커널 정보가 저장되는 디렉토리
        - `/run`: 실행 중인 서비스와 관련 디렉토리
        - `/srv`: 시스템에서 제공하는 서비스 디렉토리
        - `/tmp`: 임시 저장 디렉토리
        - `/var`: 데이터나 로그가 저장되는 디렉토리

- __사용자의 홈 디렉토리 확인__
    - 절대 경로: 루트로부터의 경로
    - 상대 경로: 현재 작업 디렉토리로부터의 경로

### 5) 파일의 구성요소

- __구성__

    - 파일 이름: 파일을 구분하기 위한 이름
        - 대소문자 구분은 한다.
    - i-node: 파일에 대한 정보
        - 파일의 종류
        - 파일의 소유권
        - 파일의 액세스 모드
        - 파일의 타임스탬프
    - 데이터 블록: 파일의 실제 내용

### 6) file

- 파일의 종류를 확인하는 명령어
- /usr/share/file 디렉토리의 magic 파일을 참조해서 파일 종류를 표시
- 형식
    - `file [옵션] [파일이나 디렉토리 경로]`

- 옵션
    - `C`: 매직 파일의 포맷을 확인하는 옵션
    - `f 목록파일`: 많은 파일을 확인할 때 사용
    - `m 매직파일`: 지정된 매직 파일로 대상 파일을 확인

### 7) 디렉토리 관련 명령

- `pwd`: 현재 작업 중인 디렉토리 확인
- `cd`
    - 다른 디렉토리로 이동할 때 사용
    - 형식: `cd [경로]`
    - 경로를 생략하면 홈 디렉토리

- `ls`
    - 디렉토리의 내용을 출력하는 명령
    - 형식: `ls [옵션] [경로]`
        - 파일을 주면 파일의 존재 여부를 확인할 수 있고 디렉토리를 주면 세부 내용을 확인하는 것이 가
    - 옵션
        - `a`: 숨김 파일도 조회
        - `d`: 디렉토리 자체 정보를 조회
        - `i`: i-node를 조회
        - `l`: 파일의 상세 정보를 조회
        - `A`: .과 ..을 제외한 정보 조회
        - `F`: 파일의 종류를 표시
        - `L`: 심볼릭 링크의 경우 원본 파일의 정보 출력
        - `R`: 하위 디렉토리까지 출력

    - 상세 정보
        - `drwxr-x--- 5 adam adam 4096 Jul 20 22:06 .`
        - `d`: 디렉토리 여부로 d이면 디렉토리
        - `rwxr-x---`: 3개씩 나누어서 해석하는데 소유자, 그룹, 기타 사용자에 대한 권한으로 r(읽기), w(쓰기), x(실행)

        - `5`: 하드 링크 개수
        - `adam`: 소유자
        - `adam`: 그룹
        - `4096`: 파일의 크기
        - `Jul 20 22:06`: 파일이 마지막으로 수정된 날짜
        - `.`: 이름
    - 디렉토리 자체 정보 확인은 `-dl`
    - 유사한 명령으로 윈도우에서 사용하던 dir과 vdir 사용 가능

- `mkdir`
    - 디렉토리를 생성하는 명령
    - 생성 권한이 있어야만 생성이 가능
    - `/tmp` 는 계정에 상관없이 생성이 가능
    - 형식
        - `mkdir [옵션] [디렉토리]`
        - 옵션은 p가 있는데 디렉토리를 만들 때 중간 경로 디렉토리가 없는 경우 중간 경로 디렉토리를 자동으로 생성해주는 옵션
            - `mkdir -p temp/mid/han`


- `rmdir`
    -  디렉토리 삭제 명령
    -  형식
        -  `rmdir [옵션] [디렉토리 경로]`
        -  비어있는 디렉토리 삭제
        -  -p 옵션을 이용하면 지정한 경로를 삭제한 뒤 상위 디렉토리들도 비어있으면 연쇄적으로 같이 삭제
        -  파일이 있으면 삭제를 못함

- 연습문제
    - 홈 디렉토리 이동
        - `cd`
    - 홈 디렉토리에 linux_ex 디렉토리를 생성
        - `mkdir linux_ex`
    - linux_ex 디렉토리에 autoever 디렉토리를 생성
        - `mkdir linux_ex/autoever`
        - `cd linux_ex && mkdir autoever`
    - autoever 디렉토리 안에 one, two, three 디렉토리를 한 번에 생성
        - `mkdir linux_ex/autoever/one linux_ex/autoever/two linux_ex/autoever/three`
    - one 디렉토리 안에 tmp/test 디렉토리를 생성하는데 하나의 명령어로 생성
        - `mkdir -p linux_ex/autoever/one/tmp/test`
    - one 디렉토리 삭제
        - `rmdir -p linux_ex/autoever/one/tmp/test`
    - two, three 디렉토리를 동시에 삭제
        - `rmdir linux_ex/autoever/two linux_ex/autoever/three`
    - 홈 디렉토리로 이동
        - `cd`

### 8) 파일 관련 명령

- `cat`
    - 텍스트 형식의 파일을 출력
    - 파일의 내용이 화면보다 크면 스크롤을 제공
    - 여러 개의 파일을 지정해서 조회 가능
    - -n 옵션을 이용해서 라인 번호 출력
    - 파일을 지정하지 않으면 입력 내용을 그대로 출력
    - /etc/hostname 파일의 내용을 출력: `cat /etc/hostname`
    - /etc/hosts 파일의 내용을 라인 번호와 함께 출력: `cat -n /etc/hosts`

- `more`
    - 텍스트 형식의 파일을 출력
    - 화면 단위로 출력하고 더 출력할 내용이 있으면 하단에 More 글자를 출력
    - `spacebar`를 누르면 다음 페이지를 출력하고 `b`를 이전 페이지를 출력
    - `/문자열`로 해당 문자열을 검색해서 이동
    - `+숫자` 옵션을 이용해서 특정 행부터 출력이 가능
    - `/etc/services`: 파일의 내용을 화면 단위로 출력: `more /etc/services`

- `less`
    - more를 개선한 것으로 방향키를 이용해서 이전에 봤던 내용을 다시 확인

- `head/tail`
    - head는 맨 앞의 10개의 행을 출력하고 숫자를 설정하면 숫자만큼 출력
    - tail은 뒤에서 출력을 함
    - tail은 -f 옵션을 이용해서 실시간으로 마지막 내용을 출력

- `cp`
    - 파일 및 디렉토리 복사
        - `cp [옵션] [파일1-디렉토리1] [파일2-디렉토리2]`
        - 파일1을 파일2에 복사
    - `-i` 옵션은 파일2가 존재하는 경우 덮어씌우는데 `-i` 옵션을 이용하면 대화식으로 작업
    - `-r` 옵션은 디렉토리를 복사할 때 사용

    - `/etc/hosts` 파일을 현재 디렉토리의 text1 파일로 복사
    - text1 파일을 temp 디렉토리로 복사: `cp text1 temp`

    - 여러 개의 파일을 복사할 때는 맨 마지막 매개변수가 파일들이 복사될 디렉토리여야 한다.

- `mv`
    - 파일의 이동과 이름 변경
        - `mv [옵션] [파일1-디렉토리1] [파일2-디렉토리2]`
        - 파일1을 파일2로 이동
    - 기존 경로가 존재하면 덮어씌움
    - `-i` 옵션을 이용하면 대화식으로 작업
    - 동일한 디렉토리 이동하면 파일 이름 변경
        - `mv ls.out ls.txt`

- `rm`
    - 파일과 디렉토리 삭제
    - 형식
        - `rm [옵션] [파일 또는 디렉토리 나열]`
    - 옵션
        - `-i`: 대화식으로 삭제
        - `-r`: 디렉토리 삭제

- `ln`
    - 링크를 생성하는 명령
    - 형식
        - `ln [옵션] <링크할 파일 경로> <링크 이름>`
    - 하드링크: i-node를 복사하는 것
        - 원본이 수정되면 같이 수정
        - 원본이 삭제되면 복사본은 유지가 됨

    - `vi print.txt` 로 파일을 만들어서 내용을 작성하고 저장
    - `ln print.txt link.ln` 명령으로 하드링크 생성
    - `vi link.ln` 으로 파일을 열어서 내용 수정
    - `cat print.txt` 로 변경되었는지 확인
    - `rm print.txt` 명령으로 print.txt 삭제
    - `cat link.ln`으로 변경되었는지 확인
