# Linux

## 1. 가상 머신에 Ubuntu 설치

### 1) Virtual Box(가상화 소프트웨어)를 다운로드 받아서 설치

### 2) Ubuntu 이미지 파일 다운로드

- Desktop: GUI 환경이 포함된 버전
- Server: GUI 환경이 포함되지 않은 버전
- GUI: 마우스나 터치패드를 이용할 수 있는 버전이고 terminal 을 열어서 사용하면 CLI 환경과 동일한 설정이 된다.
- CLI: 명령어를 직접 입력하는 방식
- CLI 환경에서 GUI를 설치
    - `sudo apt update`
    - `sudo apt upgrade`
    - `sudo apt install --no-install-recommands ubuntu-desktop`

### 3) SSH를 이용해서 외부 접속

- SSH를 설치
    - 패키지 정보 업데이트: `sudo apt update`
    - SSH 패키지를 다운로드 받아서 설치: `sudo apt install openssh-server`

- SSH 서비스 확인
    - `sudo systemctl status ssh`
    - 서비스가 active 상태가 아니라면 서비스 시작: `sudo systemctl start ssh` 

- 방화벽에서 포트 개방
    - `sudo ufw allow ssh`

- IP 확인
    - `hostname -I`

#### 포트포워딩을 수행

- NAT을 사용하는 경우
    - 가상머신을 선택한 후 설정을 클릭하고 네트워크 탭으로 이동
    - 포트포워딩을 누르고 추가
    - 호스트IP: 현재 컴퓨터의 실제 IP
    - 호스트포트: 접속하고자 하는 포트번호
    - 게스트 IP: 가상 머신에 설치된 리눅스의 IP
    - 게스트 포트: ssh의 기본 포트인 22

- 외부에서 접속
    - `ssh 계정@IP -p 포트번호`
    - 포트번호가 22번 인 경우는 -p 옵션은 생략 가능

### 4) 명령어

- GUI 환경에서는 터미널을 실행해서 작성
- 프롬프트
    - `etakyung@linuxv1:~$`
    - 모양은 변경 가능
    - super 사용자는 $대신에 #
    - etakyung은 현재 사용자 계정
    - linuxv1은 컴퓨터 이름
    - ~: 현재 디렉토리로 사용자의 홈 디렉토리

- 명령행 작성 시 발생하는 에러 대처
    - 키보드 입력이 안되는 경우: 화면을 잠그는 CTRL + s를 눌러서 그럴 수 있으므로 CTRL + q를 눌러서 화면 표시 잠금을 해제
    - 실행한 명령어가 종료가 안돼서 프롬프트가 보이지 않는 경우: CTRL + c 를 눌러서 명령 실행을 강제 종료
    - 프롬프트의 문자가 깨지는 경우에는 CTRL + I 을 눌러서 화면을 클리어

- 명령어 구조
    - `명령어 [옵션] [인자]`
    - 옵션은 세부 기능
    - -로 시작할 수도 있고 --로 시작할 수도 있다.
    - -로 시작하는 경우는 옵션이 한 글자이고 --로 시작하는 경우는 단어이다.
    - 옵션은 여러개 사용가능
    - 여러 개를 사용할 떄는 순서는 대부분 의미가 없음
    - 한 글자 옵션은 붙여서 여러 개 사용: `ls -al`

    - 인자는 명령어를 실행할 때 필요한 정보
    - 각각은 공백으로 구분
    - 명령어 도움말을 볼 때
        - []는 생력 가능
        - | 는 선택
        - 문자열, {}, <>는 필수 입력

- history
    - 위 화살표나 아래 화살표를 이용하면 이전에 실행했던 명령어를 확인하는 것이 가능
    - 명령어를 입력하고 CTRL + r 을 누르면 그 명령어로 시작하는 이전에 수행한 명령어가 조회된다.
    - 프롬프트에 history 라고 입력하면 명령어 수행 내용 출력
        - `!!`: 직전 명령어 재실행
        - `!번호`: 문자열로 시작하는 마지막 실행 명령을 실행
        - `history -d 라인번호`: 명령어 삭제
        - `history -c`: 명령어 목록 모두 삭제
        - - 명령어가 기록된 파일은 홈 디렉토리의 `.bash_history`

- `--help` 옵션
    - 사용 방법이나 옵션 등을 출력

- 명령어를 찾는 위치 -:으로 구분되어 있음
    - `echo $PATH`

- 명령어의 위치 확인
    - `whereis`: 명령어의 실행 파일 위치 및 소스 위치 등을 확인
    - `which`: PATH 환경변수에 저장된 디렉토리에서 명령어의 위치를 확인

- 기본 명령어
    - `exit`: 터미널 종료
    - `clear`: 화면 삭제

- `alias`
    - `alias`: 현재 설정된 별명을 확인
    - 설정
        - `alias 별명 = '설정값'`
    - `ls -al` 명령을 ls로 별명을 생성
        - `alias ls='ls -al'`
    - `type` 명령을 이용하면 별명인지 확인 가능
        - `type ls`
    - `unalias` 를 이용하면 별명 삭제 가능

- 원본 명령어 실행
    - 전체 경로로 명령어를 수행
    - command 명령어로 수행
    - \ 명령어로 수행

- 시간 관련 명령
    
    - `date`: Ubuntu 시스템에서 설정되어 있는 현재 시각과 날짜를 출력

    - `timedatectl`: 하드웨어 시계와 소프트웨어 시계, UTC, Timezone, 시간 동기화 등을 출력
        - timezone 목록: `timedatectl list-timezones`
        - timezone 변경: `sudo timedatectl set-timezone Asia/Seoul`


- 시스템 사용자 정보
    - logname, whoami: 현재 사용 중인 사용자를 확인
    - users, who: 현재 접속한 사용자 아이디를 전부 조회
    - uname [옵션]: 시스템 정보
        - a: 모든 정보 확인
        - m: 하드웨어 정보
        - n: 호스트 이름 확인
        - r: 운영체제 릴리즈 번호
        - s: 운영체제 이름

    - hostname: 현재 사용 중인 호스트 네임
    - arch: CPU 정보
    - env: 환경 변수

- **sudo 와 su**

    - sudo 명령어를 입력하면 관리자의 권한을 빌려서 명령을 실행하는 것으로 처음에 한 번은 관리자 비밀번호를 입력해야 함.
    - su 계정 명령어는 현재 계정의 환경 변수들을 유지하면서 다른 계정으로 전환
    - su - 계정 명령어는 현재 계정의 환경 변수를 유지하지 않고 새로운 계정의 환경 변수를 가져오면서 다른 계정으로 전환

    - su 명령만 입력하면 root 계정으로 전환

### 5) 한글 로케일 설정

- locale 확인: `locale`

- `sudo apt update`
- `sudo apt install language-pack-ko`
- `sudo update-locale LANG=ko_KR.UTF-8`
- `sudo dpkg-reconfigure locales`
- 다시 영어로
    - `sudo localectl set-locale LANG=en_US.UTF-8`

- 변경 내용은 다음 접속부터 적용되므로 exit을 눌러서 로그아웃을 하고 다시 접속하면 적용이 되어 있음

## 2. 문서 편집기

### 1) 편집기 종류

- 행 단위 편집기: `ed`, `ex`, `sed`
- 화면 단위 편집기: `vi(vim - 가장 많이 사용하는 화면 편집기)`, `emacs`, `nano(메뉴 기반 에디터)`
- GUI 편집기: `gedit`

### 2) 모드형 과 비모드형

- 모드형 편집기
    - 입력 모드와 명령 모드가 구분되어 있음

    - 입력 모드는 텍스트를 입력하고 수정할 수 있는 모드이고 명령 모드는 텍스트를 저장하거나 복사와 붙이기 같은 명령을 수행하는 모드
    - `vi`가 대표적인 모드형 편집기

- 비모드형 편집기
    - 입력 모드와 명령 모드가 구분되어 있지 않음
    - 편집 기능을 ALT 나 CTRL 같은 조합키와 사용
    - `nano`가 대표적인 비모드형 편집기

### 3) vim

- 시작
    - vi 이후에 개발된 Editor
    - 설치: `sudo apt install vim`
    - 시작: `vim [파일 경로]`
        - 해당 파일이 존재하면 파일의 내용이 보이고 존재하지 않은 파일이라면 빈 파일이 열림
        - 파일 경로를 생략하면 기본 화면이 출력됨
        - 수정 권한이 없으면 읽기 전용으로 열림

- 모드 전환
    - 명령 모드 -> 입력 모드: `i, I, a, A, o, O`
    - 입력 모드 -> 명령 모드: `ESC`


- 종료
    - 명령 모드에서 `:q, :q!, :w, :wq, :wq!`
    - w가 붙으면 저장하고 종료하고 !가 붙으면 강제 실행을 의미

- 커서 이동
    - `^`: 현재 행의 맨 앞으로
    - `$`: 현재 행의 맨 뒤로
    - `H`: 현재 화면의 맨 윗행으로
    - `M`: 현재 화면의 중간행으로
    - `L`: 현재 화면의 맨 뒷행으로

- 화면 이동
    - `CTRL + u`: 반화면 위로 이동
    - `CTRL + d`: 반화면 아래로 이동
    - `CTRL + b`: 한 화면 위로 이동
    - `CTRL + f`: 한 화면 아래로 이동
    - `CTRL + y`: 한 행만 위로 이동
    - `CTRL + e`: 한 행만 아래로 이동

- 원하는 행으로 커서를 바로 이동
    - `숫자G`: 숫자 행으로 이동
    - `:숫자[ENTER]`: 숫자 행으로 이동
    - `:$`: 파일의 마지막 행으로 이동

- 내용 수정
    - `r`: 한글자 수정
    - `cw, #cw`: 하나의 단어를 수정, # 대신에 숫자를 입력하면 숫자 단어 만큼 수정
    - `s, #s`: ESC를 입력할 떄까지 수정하는데 # 대신에 숫자를 입력하면 숫자 글자 수 만큼 수정
    - `cc`: 현재 행의 내용을 전부 수정
    - `C`: 커서 위치부터 행의 끝까지 수정

- 내용 삭제
    - `x, #x`: 글자를 삭제
    - `dw, #dw`: 단어 삭제
    - `dd, #dd`: 행을 삭제
    - `D`: 커서 위치부터 행의 끝까지 삭제

- 명령 취소
    - `u`: 명령을 취소
    - `U`: 해당 행에서 한 모든 명령을 취소
    - `:e!`: 마지막으로 저장한 내용 이후의 것을 버리고 새로 작업

- 복사, 붙여넣기, 잘라내기
    - `yy, #yy`: 커서가 위치한 행을 복사
    - `p`: 커서 아래 행에 붙이기
    - `P`: 커서 윗 행에 붙이기
    - `dd, #dd`: 잘라내기
    - 복사할 때는 버퍼이름 + yy를 사용하면 현재 행의 내용을 버퍼 이름으로 저장
    - 붙여 넣을 때는 버퍼이름 + p를 이용

- 실습
    - vim을 이용해서 exec.txt 라는 파일을 작성
    - 내용을 작성
    ```
    Good morning everyone.
    Nice to meet you.
    I am a linux begginer
    Now introduce yourself!.
    ```

- 내용 검색
    - `/문자열`: 아래 방향으로 검색
    - `?문자열`: 위 방향으로 검색
    - `n`: 검색 방향으로 다음 문자열
    - `N`: 검색 반대 방향으로 다음 문자열

- 치환
    - `:s/문자열1/문자열2`: 커서가 위치한 행에서 문자열1을 문자열2로 치환
    - `:%s/문자열1/문자열2/g`: 문서 전체에서 문자열1을 문자열2로 치환
    - %대신에 범위 설정 가능

    - 대소문자를 가리지 않고 바꾸기 수행: `set ignorecase`
    - 대문자를 소문자로 변경: `:%s/.*/\L&/`
    - 소문자를 대문자로 변경: `:%s/.*/\U&/`

- 파일 읽어오기
    - `:r 파일경로`: 지정한 파일을 읽어들여서 현재 커서 위치에 삽입
        - `:r sample.txt`
    - `:e 파일경로`: 지정한 파일로 전환을 하는데 기존 파일은 저장을 하고 실행
    - `:n`: vi 시작시 여러 파일을 지정했을 경우 다음 파일로 작업을 이동

- shell 명령 수행
    - `:! 명령`: vi 작업을 잠시 중단하고 명령어를 실행 - vim으로 돌아오려면 `Enter`
    - `:sh`: vi를 잠시 빠져나가서 명령어를 실행 - vim으로 돌아오려면 `exit`

- vim 의 환경 설정
    - 설정 방법
        - 사용자의 홈 디렉토리에 있는 .exrc(앞에 .이 붙는 파일은 숨김 파일이다.) 파일에 설정 내용을 저장
        - 환경 변수 EXINIT에 지정
        - vim의 마지막 행 모드에서 명령으로 설정
    - 환경 설정 옵션
        - `set nu`: 행 번호 표시
        - `set nonu`
        - `set list`: 눈에 보이지 않는 특수문자를 표시
        - `set nolist`
        - `set showmode`: 현재 모드를 표시
        - `set noshowmode`
        - `set`: set 으로 설정한 모든 vi환경 설정 값을 출력
        - `set all`: 모든 vi 환경 변수와 현재 값을 출력
        - `set ai`: 윗 라인과 동일한 들여쓰기
        - `set si`: 코딩을 할 때 if 나 for를 입력하면 다음 라인으로 이동할 때 자동으로 들여쓰기
    - 환경 변수를 이용해서 라인 번호 출력
        ```
        EXINIT='set nu'
        export EXINIT
        ```

        - `echo $EXINIT`: EXINIT에 할당된 환경변수를 보여줌

### 4) nano

- 개요
    - 메뉴 기반의 명령 줄 텍스트 편집기
    - 구문 강조 표시, 다중 버퍼, 검색 및 정규식 지원으로 일반 텍스트 편집기에서 기대할 수 있는 모든 기본 기능이 포함되어 있음

- 설치 및 확인
    - 확인: `nano --version`
    - 설치: `sudo apt install nano`

## 3. Shell

### 1) 개요

- 사용자가 입력한 명령을 해석을 해서 커널에 전달하고 커널의 처리 결과를 사용자에게 전달하는 역할을 수행

- Server의 테스트 모드나 GUI의 터미널처럼 명령을 입력하는 인터페이스
- 기능
    - 명령어 해석기
    - 프로그래밍 (shell programming)
    - 사용자 환경 설정
        - 초기화 파일을 제공
        - 초기화 파일에 명령을 찾아오는 경로나 파일과 디렉토리 생성시 기본 권한 설정 등을 작성
        - 사용자가 로그인할 때 설정 파일을 읽어서 사용자 별로 특성에 맞게 초기 환경이 설정됨

### 2) 종류

- 본쉘
- C쉘
- 콘쉘
- bash: 현재 리눅스에서 기본 쉘로 제공

### 3) bash shell

- 우분투에서 기본적으로 제공하는 쉘

### 4) 지원하는 쉘 확인: /etc/shells

### 5) 사용자가 로그인할 때 사용되는 쉘 확인: grep 사용자이름 /etc/passwd

### 6) 쉘을 설치하고 변경

- 설치: sudo apt install csh
- 지원하는 쉘 확인: cat /etc/shells

- 쉘 변경: 쉘은 절대 경로로 입력해야 한다.
    - chsh -s 쉘이름 계정
    - chsh -s csh adam

## 확인

- 가상환경 스프트웨어를 이용해서 리눅스 설치

- 외부에서 접속이 가능하도록 SSH를 설정하고 호스트 운영체제에서 접속
    - SSH 설치하고 서비스 시작
    - SSH를 방화벽에서 해제 (`sudo ufw allow ssh`)
    - 포트포워딩 수행
    - 호스트 운영체제의 터미널에서 접속

- 현재 설정한 운영체제의 이미지를 내보내기(SNAPSHOT 생성)
- SNAPSHOT을 이용해서 복원
- 복제
