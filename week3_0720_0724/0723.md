### 2) 패키지 관리

- 외부 저장소 이용: 도커 설치
    - 패키지 정보 업데이트: `sudo apt update`
    - 인증서 관련 패키지 설치: 

    - 도커 저장소의 인증키를 추가

- dpkg 명령으로 패키지 관리
    - 개요
        - APT 명령은 fedora 의 yun 이나 dnf 명령과 비슷한데 fedora 의 rpm 과 유사한 명령이 dpkg 명령
        - APT 도 내부적으로는 dpkg 명령을 사용
        - 일반적으로 패키지 설치는 apt를 이용하지만 시스템의 특정 파일이 어느 패키지에 속했는지를 확인하는 등 보다 세부적인 기능을 사용할 때 이용
    - 형식
        - `dpkg [옵션] 파일이름 또는 패키지이름`

    - 옵션
        - `l`: 패키지 목록을 출력
        - `s`: 상세 정보 출력
        - `L`: 파일의 목록을 출력
        - `i`: 설치
        - `r`: 삭제

- `aptitude`
    - 옵션이나 서브 명령없이 사용하면 curses를 이용한 비주얼 모드로 동작
    - 설치: `sudo apt install aptitude`
    - 확인: `sudo aptitude`

- `snap`
    - 개요
        - 우분투가 새로 도입한 패키지 형식으로 샌드박스 형태의 패키지
        - 패키지를 만들 때 필요한 모든 라이브러리를 패키지 안에 포함하는 방식
        - 외부 파일이 내부 시스템에 악영향을 주는 것을 방지
        - 단점은 패키지의 용량이 커지는 것
    - 기본 형식
        - `snap [옵션] 명령`
        - `sudo snap find hello-world`

    - 명령
        - `disable`: 스냅 서비스와 실행 파일의 사용을 중지
        - `download 스냅명`: 지정한 스냅 패키지 내려받기
        - `enable`: 스냅 서비스와 실행 파일의 사용을 시작
        - `find 스냅명`: 지정한 스냅을 검색
        - `info 스냅명`: 지정한 스냅의 상세 정보를 출력
        - `install 스냅명`: 지정한 스냅을 설치
            - `sudo snap install hello-world`
        - `list`: 설치한 스냅의 목록을 출력
        - `remove 스냅명`: 지정한 스냅을 삭제

### 3) 압축

- `tar(tape archive)`
    - 여러 개의 파일이나 디렉토리를 하나의 파일로 묶는 것
    - 형식: `tar 기능 [옵션] [아카이브파일] [파일명]`
    - 기능
        - `c`: 새로운 압축 파일을 생성
        - `t`: 압축 파일의 내용을 출력
        - `x`: 압축을 해제
        - `r`: 새로운 파일 추가
        - `u`: 수정된 파일을 업데이트
    - 옵션
        - `f`: 아카이브 파일이나 테이프 장치를 지정
        - `v`: 처리하고 있는 파일의 정보를 출력
        - `h`: 심볼릭 링크의 원본 파일을 포함
        - `p`: 파일 복구 시 원래의 접근 권한을 유지
        - `j`: bzip2로 압축하거나 해제
        - `z`: gzip으로 압축하거나 해제
    - 자주 사용하는 옵션
        - 아카이브 생성: `cvf`
        - 아카이브 압축 해제: `xvf`
        - 아카이브 내용 확인: `tvf`
        - 아카이브 업데이트: `uvf`
        - 아카이브에 파일 추가: `rvf`
    - 실습
        - 디렉토리 생성: `mkdir ex_archive`
        - 디렉토리로 이동: `cd ex_archive`
        - 디렉토리 생성: `mkdir sample1 sample2 sample3`
        - sample1 디렉토리를 압축: `tar cvf sample1.tar sample1`
        - 압축한 내용 확인: `tar tvf sample1.tar`
        - sample1 디렉토리 삭제: `rmdir sample1`
        - 압축 해제: `tar xvf sample1.tar`
        - ---
        - 파일 생성: `touch sample1/data`
        - 이미 압축된 파일을 업데이트: `tar uvf sample1.tar`
        - 압축된 내용 확인(data 라는 파일도 압축됨): `tar tvf sample1.tar`
        - ---
        - 파일 생성: `touch service`
        - 이미 압축된 파일에 추가: `tar rvf sample1.tar service`

    - 아카이브 생성하고 압축
        - gzip으로 압축하고자 하는 경우 `z` 옵션을 이용
        - 일반적으로 압축된 사실을 알려주기 위해서 마지막에 확장자 `gz`를 추가한다.
            - `tar czvf sample1.tar.gz sample1`
            - `tar czvf sample1.tar.bz2 sample1`

- `gzip/gunzip`
    - 압축률이 좋은 유틸
    - 형식
        - `gzip [옵션] [파일명]`
    - 옵션
        - `d`: 파일 압축을 해제
        - `l`: 압축된 파일의 정보를 출력
        - `r`: 하위 디렉토리를 탐색해서 압축
        - `t`: 압축 파일을 검사
        - `v`: 압축 정보를 출력
        - `9`: 최대한 압축
    - 압축된 파일의 내용을 조회: `zcat sample1.tar.gz | more`
    - 압축 해제: `gunzip sample1.tar.gz`
        - 주의할 점은 압축된 파일이 사라지면서 압축을 해제

- `bzip2/bunzip2`
    - gzip에 비하면 압축률은 좋지만 속도가 느림
        
    - 압축을 하게 되면 원본 파일이 사라진다.
        - `bzip2 sample2.tar`
        - `bunzip2 sample2.tar.bz2`

- `xz`나 `zip`도 지원

### 4) wget

- 웹 상의 파일을 다운로드 받을 때 사용하는 명령어
- 형식
    - `wget [옵션] [URL]`
- 옵션
    - 일반 다운로드: `wget 다운르도 받을 파일의 URL`
    - 다른 이름으로 저장: `wget -O 파일명 다운로드 받을 파일의 URL`
    - 이어서 받기: `wget -c 다운로드 받을 파일의 URL`
    - 백그라운드에서 다운로드 받기: `wget -b 다운로드 받을 파일의 URL`
    - 속도 제한: `wget --limit-rate=속도 다운로드 받을 파일의 URL`

- 웹사이트 미러링
    - `wget -m 웹사이트URL`
    - `-r`: 하위 디렉토리를 따라가며 다운로드
    - `-np`: 상위 디렉토리로 이동하지 않도록 제한
    - `-k`: 로컬에서 오프라인으로 보기 편하도록 링크를 반환

### 5) curl

- 서버와의 데이터 통신 및 API 테스트에 최적화되어 있어서 개발자들이 선호
- 웹 페이지 소스 보기: `curl URL`
- 파일로 저장
    - 동일한 이름으로 저장: `curl -O 파일경로`
        - `curl -O https://www.naver.com/index.html`
    - 이름을 변경: `curl -o 파일명 파일경로`
        - `curl -o default.html https://www.naver.com/index.html`

- 헤더 확인: `curl -l https://www.naver.com`
- 리다이렉트 자동 추적: `curl -L https://www.naver.com`
- API 테스트할 때 사용하는 옵션
    - `-X`: HTTP 메서드 설정
    - `-d`: 전송할 데이터를 생성
    - `-H`: 헤더 정보 설정

- 자주 사용하는 옵션: `fsSL`
    - `f`: 오류가 발생하는 경우 실패 처리
    - `s`: 에러 메시지 표시하지 않음
    - `S`: 진짜 에러인 경우만 출력
    - `L`: 리다이렉트 되는 경우 자동으로 새로운 주소로 가서 데이터를 가져온다.
    - 다운로드 받아서 설치할 때 주로 이용
        - `curl -fsSL https://install.direct/go.sh | sudo bash`

### 3) systemd service

- 리눅스 시스템의 서비스 관리자
- 서비스 데몬을 시작하고 프로세스의 상태를 유지하면 시스템의 상태를 관리
- init 프로세스
    - 이 프로세스가 다른 프로세스들을 실행
    - 이 프로세스의 스크립트 디렉토리는 `/etc/init.d`
- `systemd unit`
    - 전체 시스템을 시작하고 관리하는데 유닛이라 부르는 구성 요소를 사용
    - 관리 대상의 이름을 `서비스명.유닛종류` 로 관리
        - 부를 때는 유닛종류를 생략하는 것이 일반적
    - 유닛 종류
        - `service`: 시스템 서비스 유닛으로 데몬을 시작, 종료, 재시작, 로드
        - `target`: 유닛을 그룹화
        - `automount`: 디렉토리 계층 구조에서 자동 마운트 포인트를 관리
        - `device`: 리눅스 장치 트리에 있는 장치 관리
        - `mount`: 디렉토리 계층 구조의 포인트 관리
        - `path`:
        - `scope`: 외부에서 생성된 프로세스 관리
        - `slice`: 시스템의 프로세스를 계층적으로 관리
        - `socket`
        - `swap`: 스왑 장치 관리

- `systemctl 명령어`
    - 형식: `systemctl [옵션][명령][유닛이름]`
    - 옵션
        - `a`: 상태와 관계없이 유닛 전체를 출력
        - `t 유닛종류`: 유닛 종류에 해당하는 것만 출력
    - 명령
        - `start`: 시작
        - `stop`: 중지
        - `reload`: 설정 파일을 다시 읽어오기
        - `restart`: 재시작
        - `status`: 상태 확인
        - `enable`: 부팅하자마자 서비스를 시작
        - `disable`: 부팅하자마자 서비스를 시작하지 않도록
        - `is-active`: 시작되었는지 확인
        - `is-enabled`: enable 여부
        - `isolate`: 지정한 유닛만 시작하고 나머지는 정지
        - `kill`: 유닛에 시그널을 전송

    - cron 서비스를 시작: `sudo systemctl start cron`
    - 연습
        - `colord.service` 가 동작 중인지 확인
            - `systemctl is-active colord.service`
        - 동작 중이면 중지
            - `sudo systemctl stop colord.service`
        - 시작
            - `sudo systemctl start colord.service`
        - PID 확인
            - `systemctl show -p MainPID colord.service`

- `RUN level`
    - 현재 시스템의 상태를 나타내는 한 자리 숫자
        - `0`: poweroff
        - `1`: rescue 모드 - 안전 모드
        - `2`: 다중 사용자
        - `3`: 다중 사용자
        - `4`: 다중 사용
        - `5`: GUI
        - `6`: 재부팅

    - 수정: `sudo systemctl set-default runlevel숫자.target`
    - 기본은 5번을 선택하는 것을 권장하고 부팅이 제대로 되지 않을 때는 1번으로 수정해서 확인
        - 1번으로 부팅하면 GUI 환경이 아니라 텍스트 모드로 부팅된다.

- 서비스 등록
    - 서비스 파일을 작성: /etc/systemd/system 디렉토리에 .service 파일을 생성
    - 파일 내용
        - `[Unit]`
            - `Description=설명`
            - `After=어떤 서비스를 먼저 실행해야 하는지 설정`
        - `[Service]`
            - `ExecStart=실제스크립트파일경로`
            - `Restart=재시작옵션(on-failure)`
        - `[Install]`
            - `WantedBy=런레벨타겟`
                - 파일을 작성한 후 서비스를 새로 읽어내도록 reload: `sudo systemtl daemon-reload`

### 5) Namespaces & Cgroup

- Container
    - 컨테이너 기술은 애플리케이션을 효율적이고 독립적으로 실행할 수 있는 경량화된 환경을 제공하는 것
    - 기본적으로 애플리케이션을 실행하려면 운영체제와 애플리케이션 그리고 필요한 라이브러리가 함께 제공이 되어야 하는데 컨테이너는 애플리케이션이 실행하기 위해 필요한 최소한의 운영체제 기능을 가지고 애플리케이션을 실행
    - 컨테이너의 근간이 되는 리눅스 기술
        - `Cgroup`
        - `Namespaces`
        - `Union Mount Filesystem`
    - `Cgroup(control group)`
        - 프로세스들이 사용하는 시스템 자원의 사용 정보를 수집 및 제한하는 기능
        - 사용 가능한 서브 시스템
            - `CPU`
            - `Memory`
            - `Freezer`: cgroup의 작업을 일시 중지하거나 다시 시작
            - `blkio`: 입출력 제한 설정
            - `net_cls`: cgroup 작업에서 발생하는 패킷을 식별하게 하는 네트워크 패킷 태그를 작성
            - `cpuset`: CPU를 cgroup에 할당
            - `cpuact`: CPU 자원 보고서
            - `devices`: 작업 별로 장치에 접근을 제어
            - `ns`: 네임스페이스
        - /sys/fs/cgroup 디렉토리에 가상 파일로 존재하고 해당 파일들에 대한 설정들을 변경해가면서 사용 가능
        - cpu 사용 제한
            - /sys/fs/cgroup 디렉토리 안에 별도의 디렉토리를 생성
            - 생성한 디렉토리에 파일을 생성해서 작성
            - 사용시간 100000 이렇게 작성하면 100ms 중 사용시간 만큼 사용을 하게 된다.
            - 전체 시간을 전부 사용하고자 하면 max 라고 적으면 되고 그 이외는 비율로 설정
                - 절반 사용: 50000 100000
                - 전체 사용: max 100000
    - `namespace`
        - 프로세스를 격리시키기 위한 논리적인 그룹
    - `Union Mount Filesystem`
        - 프로세스 별로 별도의 파일을 소유해서 마운트 시킨 후 사용

## 디스크 관리

### 1) 디스크 기반 파일 시스템

- 개요
    - 파일과 디렉토리를 관리하려면 파일 시스템이 필요
    - 초창기에는 Minix File System을 사용했는데 최근에는 ext 라는 리눅스 고유의 파일 시스템을 만들어 사용
        - 지금은 ext4까지 제공하고 있는데 대용량 파일 시스템을 위한 XFS 파일 시스템도 도입됨

- 종류
    -`ext4`
        - 1EB(1024 * 1024 TB) 까지의 디스크 크기를 제공하고 16TB 까지의 파일을 지원
        - 디렉토리 개수는 64000개 까지 가능
    - `XFS`
        - xfsprogs 패키지를 설치해야 사용 가능
        - 16EB 까지 가능

- 리눅스에서 지원하는 파일 시스템
    - `nfs`: 원격 서버의 디스크를 연결할 때 사용
    - `ntfs`: 윈도우 파일 시스템
    - `hfs`: Mac 파일 시스템

- 특수한 용도의 파일 시스템
    - `swap`: 가상 메모리 사용을 위한 파일 시스템
    - `tmpfs`: 메모리에 임시 파일을 저장하기 위한 파일 시스템인데 시스템이 재시작하면 내용이 소멸되는데 /tmp 디렉토리가 대표적
    - `ramfs`: 디스크 대신 메모리를 저장공간처럼 사용하는 파일 시스템으로 임베디드 장치에서 사용하는데 tmpfs와 다른 점은 swap 사용 여부로 tmpfs는 필요시 swap 사용
    - `rootfs`: 시스템 초기화 및 관리에 필요한 내용을 저장하고 있는 루트 파일 시스템
    - `proc`
        - 커널의 현재 상태를 나타내는 파일을 소유하며 실
        - `/proc/cpuinfo`

### 2) 리눅스에서 새로운 디스크 사용 방법

- 디스크 장착

- 디스크 확인: `sudo fdisk -l`
    - 디스크는 sd영문자 형태로 이름이 만들어지는데 sda sdb sdc 형태로 만들어진다.

- 파티션(데이터를 기록할 수 있는 논리적인 공간) 생성
    - `sudo fdisk /dev/하드디스크이름`

    - 새로 만들 때는 `n`
    - 기존 파티션 삭제는 `d`

    - 생성해야 하므로 `n`
    - 파티션 종류 선택: 
        - Primary(4개까지 가능) 와 Extended
    - 파티션 번호: 1~4 사이

    - `First Sector`: 시작위치로 기본은 2048부터
    - 이를 이용해서 하나의 물리적인 디스크를 논리적으로 여러 개 만들 수 있다.

    - `Last Sector`: 기본값은 최대값(사이즈나 섹터 번호를 설정)
    - 하나의 파티션이 생성됨
    - 마지막에 w를 입력을 해야만 실제로 파티션을 만듦

- 포맷 및 파일 시스템 생성

    - `sudo mke2fs -t ext4 /dev/sd디스크이름파티션번호`
    - `sudo mke2fs -t ext4 /dev/sdb1`

- 마운트
    - `sudo mount /dev/sd디스크이름파티션번호 마운트할디렉토리`

- virtual box 같은 가상화 소프트웨어에서 디스크 크기를 늘린 경우 이 작업을 수행을 해야 한다.

### 3) LVM (중요) - Logical Volume Manager

- 개요
    - 리눅스의 저장 공간을 효율적이고 유연하게 관리하기 위한 커널의 한 부분

    - 여러 개의 하드디스크를 한 개의 파티션으로 구성한 후 다시 필요에 따라 나누는 것
    - Volume 이라는 개념을 도입해서 저장 장치를 다룸
    - 스토리지에 확장이나 변경 시 유연하게 크기를 변경할 수 있고 기존 데이터의 이전이 필요없음

- 기본 개념
    - PV: 실제 하드디스크의 파티션
    - VG(Volume Group): 여러 개의 PV를 묶은 것
    - LV(Logical Volume): VG를 다시 적절한 크기의 파티션으로 분할한 것
    - PE: PV가 가진 일정한 블록
    - LE: LV가 가진 일정한 블록

- 과정
    - fdisk 명령을 이용해서 파일 시스템 생성
    - pvcreate 명령으로 pv를 생성
    - vgcreate 명령으로 pv를 하나의 vg로 생성
    - vgchange -a y 명령으로 vg를 활성화
    - lvcreate 명령으로 lv를 생성
    - mke2fs 명령으로 lv에 파일 시스템 생성
    - lv를 마운트해서 사용

- 클라우드 환경의 저장소 제공 서비스에서 이 기능을 사용

### 4) 디스크 사용량 확인

- 파일 시스템 별로 디스크 사용량 확인: `df`
    - 형식: df [옵션] [파일시스템]
    - 옵션
        - `a`: 모든 파일 시스템 조회
        - `k`: KB 단위
        - `m`: MB 단위
        - `h`: 알기 쉬운 단위
        - `t 파일시스템종류`: 파일 시스템 종류에 해당하는 것만 조회
        - `T`: 파일 시스템 종류를 출력

- 디렉토리나 사용자 별 사용량 확인: `du`
    - 형식: `du [옵션] [디렉토리]`
    - 옵션
        - 옵션없이 사용하면 현재 디렉토리의 사용량
        - `-s 디렉토리`: 디렉토리의 사용량
        - `-h`: 알기 쉬운 단어로 출력

### 5) 파일 시스템 검사 및 복구

- 파일 시스템 검사: `fsck`

    - 형식: `fsck [옵션] [장치이름]`
    - 옵션
        - `f`: 강제로 점검
        - `b`: 슈퍼블록으로 지정한 백업 슈퍼블록을 사용
        - `y`: 모든 질문에 yes로 대답
        - `a`: 문제가 생겼을 때 자동으로 복구

- 디스크 검사: `e2fsck`
- 배드블록 검사: `badblocks`

### 6) RAID

- 개요
    - Redundant Array of Independent Disks는 여러개의 하드 디스크를 하나의 하드 디스크처럼 사용하는 방식
    - 비용을 절감하면서도 신뢰성을 높이며 성능까지 향상 가능
- 방식
    - Hardware RAID
        - 하드웨어 제조업체에서 여러 개의 하드 디스크를 연결한 장비를 만들어 그 자체를 공급하는 것
        - 고가의 경우 SA-SCSI 방식을 사용하고 중저가의 경우는 SATA 하드 디스크를 이용

    - Software RAID
        - 여러 개의 하드 디스크를 묶어서 운영체제에서 지원

- 기대효과
    - 대용량의 단일 볼륨을 사용하는 효과
    - 디스크 I/O 병렬화로 인한 성능 향상
    - 데이터 복제로 인한 안정성 향상: 고가용성(원하는 경우 언제든 사용 가능해야 한다)

- RAID 레벨: Linear, 0, 1, 2, 3, 4, 5
    - `Linear RAID`: 여러 개의 하드 디스크를 연결해서 첫번째 하드 디스크부터 순차적으로 사용
    - `RAID0`: 여러 개의 하드 디스크를 연결해서 병렬로 사용 가능(속도가 가장 빠름), stripping
    - `RAID1`: 2개의 하드 디스크에 동일한 내용을 저장, 결함 허용 시스템을 구성, 미러링
    - `RAID5`: 3개 이상의 하드 디스크를 가지고 하나는 오류를 검출하기 위한 디스크로 사용
    - `RAID6`: 4개 이상의 하드 디스크를 가지고 오류를 중첩해서 검출
